Microsoft 365 Copilot App · 总入口

Windows 任务栏 · 浏览器 m365.cloud.microsoft · 移动端 · 官方页：support.microsoft.com/Microsoft-365-Copilot

The Copilot App 是进入整个 Copilot 世界的统一大门。不是"又一个聊天工具"——它连接你的工作数据（邮件、文件、会议、联系人），给出的回答基于你的实际工作上下文。

Chat

跨应用工作问答——搜邮件、找文件、问会议，一个对话窗口搞定

Notebooks

多文件深度研究——把散落资料放进一个 Notebook 综合分析

Agents

自动化助手——创建专属 Agent 处理重复任务

关键认知：Copilot App 与 ChatGPT 的根本区别——ChatGPT 只知道互联网上的公开信息，Copilot App 知道你上周跟谁开了会、你的 OneDrive 里有什么文件、你收到了哪些邮件。工作上下文是 Copilot 的核心竞争力。

Copilot Chat · 跨应用工作问答

所有人的每日入口 · 搜邮件 + 找文件 + 问会议 + 起草内容 · 一个对话窗搞定

Copilot Chat 是最高频、最基础的 Copilot 使用方式。它连接你的 Microsoft 365 工作数据——邮件、文件、会议、联系人——像一个了解你工作全局的助手。

信息检索型

"找文件""查邮件""问会议"——Copilot 跨 Outlook/OneDrive/SharePoint/Teams 检索，不需要知道文件存在哪里。

总结归纳型

"总结本周邮件""提炼三场会议的共性"——从大量信息中提取要点，省去逐条阅读时间。

创作生成型

"起草一封邮件""写一份简短报告"——基于你的工作上下文生成内容，不是泛泛的模板。

分析推理型

"比较两个方案""分析这个趋势背后的原因"——结合多个数据源进行交叉分析。

检索
"帮我找上周张三发的那份合同。顺便告诉我合同金额和到期日期。"
Copilot 搜索邮件+OneDrive，直接提取关键信息
总结
"总结过去一周我收到的所有关于 Q3 预算的邮件，列出关键变化和建议。用表格。"
创作
"基于今天讨论的会议纪要，给客户写一封跟进邮件。语气专业但不生硬。"
分析
"比较一下 Q2 和 Q3 的供应商报价，找出价格变化最大的三项，分析可能的原因。"
关键技巧：Chat 的能力上限取决于你给它的上下文。模糊的问题得到模糊的答案——"帮我看下合同"远不如"帮我找出跟XYZ公司签的那份服务合同，看看续约条款"。
来源：Microsoft Support · Get instant answers with M365 Copilot Chat (2026-02)

Outlook · 邮件处理

读邮件 · 写回复 · 调语气 · 排日程 · 每天至少省 30 分钟

Outlook Copilot 覆盖邮件的完整处理流程——从读、到理解、到回复、到跟进。

快速阅读

20 段长的客户邮件→点 "Summary by Copilot" → 5 秒出 3 条要点。不用逐段读。

智能起草

告诉 Copilot 你想说什么→它起草回复。可选语气：正式/友好/简洁/热情。

邮件线程总结

20 封来回的邮件线程→"总结这个讨论，各方立场是什么，最后达成了什么共识。"

会邀与跟进

"根据这封邮件的内容，帮我安排一个 30 分钟的跟进会议，附上讨论要点。"

多指令回复
"回复这封邮件：感谢建议，同意前两点，第三点需再讨论——提出两个替代方案。语气礼貌但坚定。在回复开头附上原邮件的三点摘要。"
Copilot 能同时处理多个指令——不是只做一件事
语气切换
"把这段回复的语气改得更正式——这是给客户的。再生成一个内部版本，语气直接一些。"
日程安排
"从这封邮件中提取所有提到的时间节点和截止日期，帮我创建对应的日历提醒。"
关键技巧：只说"帮我回复"=模糊=结果随机。好指令=给立场 + 给结构 + 给语气——"帮我回复，表达感谢+同意前两点+对第三点提替代方案，语气礼貌但坚定。"

Teams · 会议与协作

会前准备 · 会中实时 · 会后复盘 · 聊天总结 · 不只是"补会议"

Teams 里的 Copilot 覆盖会议全生命周期——不只是追错过的东西。它是日常协作中最高频的 Copilot 入口。

会前

在会议聊天中提前问 Copilot："这次会议要讨论什么？有哪些材料我需要提前看？"——Copilot 会读取会议邀请、附件和会前 24 小时内的聊天记录。

会中

实时提问："目前讨论了几种方案？""刚才谁提了关于预算的问题？"——不需要记笔记，Copilot 帮你捕捉关键瞬间。即使未开启转录也能使用基础功能。

会后 · Recap 页

如果开启了转录，从 Recap 标签页访问 Copilot——获取带时间戳的摘要、谁说了什么、智能提取的行动项。（Intelligent Recap 需 Teams Premium 或 M365 Copilot 许可证）

聊天与频道

在任意聊天中让 Copilot 总结未读消息、提取关键决策、改写消息语气。频道中可添加 Channel Agent 自动回答常见问题。

会前准备
"总结这次会议要讨论的内容，列出我需要提前看的所有材料。"
Copilot 读取会议邀请 + 附件 + 会前聊天记录
会中实时
"目前讨论了哪几个方案？各自的优缺点是什么？有没有还没被提到的重要风险？"
错过会议
"总结我刚错过的这个会。列出所有决定、行动项和负责人。用表格，附时间戳。"
多会议回顾
"总结本周我参加的三场客户会议的共同主题和待跟进事项。"
需 Teams Premium — Multi-meeting audio recaps 功能

Word · 文档起草与改写

从零起草 · 基于模板 · 改写润色 · 长文总结 · 告别"空白页恐惧症"

Word Copilot 覆盖文档的全生命周期——从第一个字到最终版本。

从零起草

打开空白文档→告诉 Copilot 主题+结构+字数→生成初稿。再也不盯空白页发呆。

基于现有文件

引用已有文档为新版本的基础——"基于 Q2 报告写 Q3 版，保留结构，更新数据。"

改写与润色

"改得更正式""缩短到 500 字""换个更积极的角度"——同一内容多版本输出。

长文理解

30 页报告→"总结核心论点""列出所有数据来源""找出与其他文档的矛盾之处"。

结构化起草
"起草一份 Q3 销售回顾报告：①总体业绩（含同比环比）②区域对比 ③产品线分析 ④Q4 展望。语气正式，800-1000 字。附一个 Executive Summary。"
版本迭代
"基于 /Q2销售报告.docx 写 Q3 版本——保留所有章节结构，用以下新数据替换：华东增长 12%、华南下降 3%、新品线贡献 18%。"
多版本输出
"把这份技术白皮书改写成三个版本：①给高管的 1 页摘要 ②给客户的 3 页商务版 ③给工程师的完整版。"
深度分析
"阅读这份 50 页行业报告，提取与我们业务最相关的 5 个趋势，每个附原文引用和我们的应对建议。"
关键技巧：Word Copilot 的强大在于"引用现有文件"能力——不是从零生成，而是基于你的真实文档做版本迭代。给一个文件路径比给一段描述效果好得多。
来源：Microsoft Support · Draft and add content with Copilot in Word | Summarize a document with Copilot

PowerPoint · 演示文稿

一句话出 PPT · Word→PPT · 设计调整 · 图片生成 · 告别空白页

PowerPoint Copilot 让你跳过"新建空白演示文稿"的痛苦。从大纲到完稿，从设计到演讲备注。

从零生成

告诉 Copilot 主题+页数+风格→生成完整 PPT 初稿，含标题页、内容页、图片和演讲备注。

从文档生成

给一份 Word 大纲→"基于这个文档生成 10 页 PPT，每页一个主题。"保留所有数据和图表。

设计调整

"把蓝色主题换成公司品牌色""第 5 页的图表换成柱状图""给所有页面加上公司 Logo 水印"。

演讲辅助

"给每页生成演讲备注""把第 3 页的内容提炼成 3 个 talking points""给这 20 页 PPT 生成一个 Agenda 页"。

主题生成
"创建一份关于数字化转型的 10 页 PPT：①现状分析②技术路线图③实施计划④预期收益⑤风险与应对。风格简洁专业，蓝色主题。"
文档转PPT
"基于 /Q3业务回顾.docx 创建 PPT，每页一个章节，保留所有数据图表，加上一页 Executive Summary。"
快速迭代
"把第 7 页拆成两页——一页讲数据，一页讲结论。给所有图表统一加上数据来源标注。"
演讲准备
"给每页生成 3 个演讲要点。再生成一页 Q&A 预判——列出客户可能问的 5 个问题及建议回答。"
关键技巧：PPT Copilot 的生成质量 = 输入的详细程度。只说"做个 PPT"=生成泛泛的模板。好输入=明确页数 + 每页主题 + 风格要求 + 数据来源。
来源：Microsoft Support · Create a new presentation with Copilot in PowerPoint | Add a slide or image with Copilot

Excel · 数据分析

自然语言替代公式 · 自动图表洞察 · 条件格式 · 让非技术人员也能做分析

Excel Copilot 把"数据分析"从专业能力变成通用能力。你不需要知道 VLOOKUP 或 PivotTable——告诉 Copilot 你想要什么结果。

探索与洞察

"分析这个数据表""找出异常值""哪个产品线增长最快"——Copilot 自动生成图表+文字洞察。

公式与计算

"加一列同比增长率""计算加权平均""按区域分组求和"——自然语言替代公式。

可视化

"用柱状图展示区域对比""加一条趋势线""高亮增长率超过 20% 的行"——自动生成图表和条件格式。

假设分析

"如果价格下降 10%，对利润的影响？""各区域的盈亏平衡点在哪里？"——让数据回答"what if"。

趋势分析
"分析这个销售数据表：①找出增长最快和最慢的三个产品线 ②用图表展示月度趋势 ③解释可能的原因 ④给出 Q4 建议。"
异常检测
"扫描这份财务报表，找出所有与去年同期偏差超过 15% 的科目，高亮标记并解释可能原因。"
批量计算
"在表里加三列：同比增长率、环比增长率、YTD 累计。高亮所有负增长行。"
What-If
"如果原材料成本上涨 8%，各产品线的毛利率变化如何？生成一张敏感性分析表。"
限制说明：Copilot 在 Excel 中需要数据是结构化表格格式（Ctrl+T）。如果数据是散乱的非表格格式，需先整理。另外 Excel Copilot 目前仅支持英文 Prompt，中文支持正在推进。
来源：Microsoft Support · Get started with Copilot in Excel

OneNote · 笔记整理

笔记总结 · 待办提取 · 知识整合 · 让散乱笔记变结构化文档

OneNote Copilot 解决一个经典痛点：写了很多笔记，但从不再看。它把散乱的手写、打字、截图笔记变成可检索、可行动的结构化内容。

总结与提炼

"总结这几页会议笔记""提取所有讨论过的方案和各自优缺点"——把几页手写变几行要点。

待办提取

"列出所有待办事项，按优先级排序，标注负责人和截止日期"——从散乱笔记中提取 Action Items。

知识整合

"把这三页关于新员工入职流程的笔记整合成一个 SOP 文档"——多页笔记→一份结构化文档。

创意展开

"基于这些产品想法，展开每个想法的优劣势和可行性""帮我把这些零散点子组织成一份提案大纲"。

会议笔记
"总结这几页会议笔记：①讨论了哪几个议题 ②各方立场是什么 ③达成什么共识 ④列出所有行动项（负责人+截止日期）。"
知识归纳
"把过去三个月所有关于'客户投诉处理'的笔记整合成一份最佳实践文档，包含：常见问题分类、处理流程、话术模板。"
项目复盘
"整理这个项目的所有笔记：时间线、关键决策节点、遇到的问题和解决方案、经验教训。"

OneDrive / SharePoint · 文件知识发现

对文件提问 · 跨文件对比 · 公司知识检索 · Copilot 最被低估的能力

你的文件不再沉默。OneDrive 和 SharePoint 里的成百上千个文件——合同、报告、政策——Copilot 能读懂它们，让你用自然语言提问。这是 Copilot 最被低估的能力：不是帮你"写"，而是帮你"找到"和"理解"。

单文件深度问答

"这份 80 页的合同里，违约责任条款在哪里？如果我方延迟交付，最大赔偿是多少？"

跨文件对比分析

"比较这三家供应商的报价：价格差异最大的三项、各自的交期承诺、付款条件对比。做成表。"

公司知识检索

"公司关于远程办公的政策是什么？海外出差审批流程是什么？"——不用翻遍 SharePoint。

历史追溯

"这个客户过去三年签了哪些合同？合同金额趋势如何？有没有出现过变更或纠纷？"

合同分析
"这份合同：①违约条款怎么说 ②甲方延迟付款超 30 天有什么后果 ③自动续约条款存在吗 ④相比我们标准条款模板，有哪些不利条款？"
跨文件对比
"对比去年和今年的供应商报价——哪家价格变化最大？有没有新增供应商？用表格列出 Top 5 价格变动。"
政策查询
"公司关于出差报销的最新政策是什么？国外出差和国内出差的报销标准和审批流程有什么区别？"
关键认知：这个能力的前提是你的文件存放在 OneDrive/SharePoint 且你有访问权限。本地文件、邮件附件未保存到云端、或权限外的文件，Copilot 无法读取。
来源：Microsoft Support · OneDrive Copilot | SharePoint Copilot · Ask questions about files

Copilot Notebooks · 多资料深度研究

多文件综合分析 · 交叉验证 · 研究助理 · Copilot 的"深度模式"

Notebooks 是 Copilot 的"研究模式"——不同于 Chat 的快速问答，Notebooks 让你把多份不同类型文件放进去，做交叉分析和深度洞察。

多源整合

把 Word/Excel/PPT/PDF/邮件放进一个 Notebook → 集中提问。支持最多几十个文件同时分析。

交叉验证

"这三份市场报告的数据有矛盾吗？哪个数据源最可信？"——AI 帮你找 inconsistency。

结构化输出

"把这五份客户反馈整合成一份 SWOT 分析""把所有数据汇总成一张对比表"——多文件→一份结构化文档。

研究加速

竞品分析、市场研究、文献综述、行业扫描——把原来需要几天的资料整理压缩到几小时。

竞品分析
"把这三份竞品报告+两份市场数据整合分析：①我们的 Top 3 竞争优势 ②最大的 2 个威胁 ③建议的差异化策略。"
数据交叉验证
"这四份行业报告引用的市场规模数据——哪些互相矛盾？差异的原因是什么？哪个数据源最可靠？"
方案对比
"把这三份 IT 系统选型方案做对比：功能覆盖、实施周期、总成本、风险——输出一份对比评估表，附推荐排序。"

"阅读这 10 篇关于 ISO 45001 更新的文章，提取所有新要求、变化点和实施建议。按主题分类输出。"
与 Chat 的区别：Chat 适合快速单问题（"找一份文件""总结一封邮件"），Notebooks 适合需要多文件上下文、持续追问、深度分析的场景。Notebooks 中的对话有"记忆"——前面问过的上下文会在后续问题中保留。

AI 内容生成 · 图片 / 视频 / 信息图

图片生成

短视频

"生成一段 30 秒的产品介绍短视频"——适合社交媒体、内部培训、客户简报。

图片
"生成一张 TÜV 莱茵认证审核场景的配图，用于培训 PPT 封面。风格：专业、现代、蓝白色调。"

Video / Clipchamp · 视频理解

对视频提问 · 时间戳跳转 · 自动摘要 · 让视频像文档一样可检索

视频不再需要从头看到尾。Copilot 能理解视频内容——培训视频、产品演示、会议录播——让你用自然语言提问，直接跳到关键段落。

内容检索

"这个培训视频讲了哪些关键概念？""有没有提到 ISO 27001 的新要求？"——不用快进，直接问。

时间戳跳转

"从哪个时间点开始介绍新功能？""谁在什么时候提到了预算问题？"——附带可点击的时间戳。

批量处理

"总结本周三场培训视频的共通主题""哪场提到了数据合规？"——跨视频检索。

Clipchamp 编辑

在 Clipchamp 中可以直接用 Copilot 理解视频内容并辅助编辑——剪出关键片段、添加字幕。

培训视频
"总结这个 45 分钟的培训视频：①讲了哪几个模块 ②每个模块的关键概念 ③最后 10 分钟的 Q&A 涉及哪些问题。"
产品演示
"这个产品演示视频里，新功能从哪个时间点开始介绍？演示了什么场景？有没有提到已知限制？"
多视频对比
"比较这两场客户演示的录制视频——哪个演示风格更有效？客户问了什么共同问题？"
适用场景：培训视频复习、客户演示回顾、合规审计视频检索、会议录制追看。视频需上传到 OneDrive/SharePoint 或通过 Clipchamp 处理——本地未上传的视频无法被 Copilot 理解。

Agents in SharePoint · SharePoint Agent

Microsoft Adoption · adoption.microsoft.com/ai-agents/agents-in-sharepoint · 下载 2026-07-14

SharePoint Agent 是最"低门槛高价值"的 Agent 类型——不需要任何技术能力，用户选择已有 SharePoint 站点作为知识源，Agent 就能基于站点内所有文档、列表和页面回答问题。

莱茵适用场景
  • 认证标准库 — ISO/行业标准的条款检索和解释
  • 项目知识库 — 某个客户项目的所有文档、邮件、会议纪要
  • 部门 SOP — 操作手册、流程规范、模板文档
  • 竞品情报 — 市场分析报告、竞对信息的集中检索
创建只需 3 步
  1. 在 Copilot Chat 中选择 "Create an agent"
  2. 指定一个 SharePoint 站点作为知识来源
  3. 给 Agent 命名 + 简短描述（如"莱茵 ISO 9001 标准助手"）

创建后立即可用——团队成员在同一 Copilot Chat 中 @Agent名称 即可提问。

注意事项：Agent 只能访问用户已有权限的文档。如果一个站点包含敏感信息，需先通过 SharePoint Governance 限制访问范围。Agent 创建后可通过 Agent Registry 集中管理。

适用对象：种子用户 · 业务团队 · 知识管理员 · 项目经理 · SharePoint 管理员

Employee Self-Service Agent · 员工自助服务

Microsoft Adoption · adoption.microsoft.com/ai-agents/employee-self-service · 下载 2026-07-14

这是莱茵内部 Agent 落地最直接、ROI 最可量化的应用场景。不需要复杂技术——只需将现有政策文档、FAQ、流程说明作为 Agent 的知识源，员工就能用自然语言自助查询。

典型知识源
  • HR 政策 — 假期、报销、差旅、福利、入职离职流程
  • IT 支持 — 账号申请、VPN 配置、软件安装、打印机
  • 行政服务 — 会议室预订、访客登记、办公用品
  • 合规问答 — 数据保护、反腐败、安全规范
莱茵落地步骤
  1. 选择一个部门试点（HR 或 IT 支持）
  2. 整理该部门的 Top 20 高频问题 + 对应文档
  3. 用 Agent Builder 创建 Agent，指定 SharePoint 站点为知识源
  4. 在部门内测 2 周，收集反馈和缺失问题
  5. 迭代知识库 → 扩展到其他部门

关键成功因素：Agent 的质量 = 源文档的质量。如果政策文档本身混乱、过时或缺失，Agent 无法给出好答案。部署 Agent 之前先做一次文档审计和更新。

适用对象：HR · IT 支持 · 行政 · 部门负责人 · Champion/种子用户
