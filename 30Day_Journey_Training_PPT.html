三种执行方式

方式一：Journey Kit

自建邮件序列

下载可定制的邮件模板（.OFT 格式）本地化通过公司邮件系统发送。适合有 IT 支持的组织。当前可用。

方式二：Microsoft 直发

Microsoft 代发邮件

管理员在 Admin Center 开启Microsoft 直接向终端用户发送 6 封周报邮件。零配置。完整版 2025.3 可用，Chat 版 2025 夏可用。

方式三：线上培训活动

Virtual Training Event

参加或邀请用户参加 Microsoft 官方的线上培训——获取灵感和技巧。适合作为 Journey 的补充。

莱茵建议：优先方式一（Journey Kit）——完全掌控邮件内容、发送节奏和本地化。方式二作为补充（管理员开启后 Microsoft 自动发，零维护成本）。方式三适合 Journey 结束后持续学习。

5 步实施框架（Pilot Phase）

Source: Overview PDF Page 5 "The Process (pilot phase)"

01

Setup
准备

• 定义目标人群和启动时间
• 下载 Kit + 定制邮件
• 设置专用邮箱
• 预排邮件发送

02

Kick-Off
启动

• 领导发送欢迎信
• 发送 Journey 前问卷
• 发送第一封邮件
• 测量 DAU 基线

03

Daily Tasks
日练

• 用户每天完成 1 个 Prompt
• 监控邮件打开率
• 监控日活 (DAU)

04

Closing
结业

• 最后一封邮件 + Badge
• Journey 后问卷
• 鼓励分享 Badge
• 驱动进一步采纳

05

Follow Up
跟进

• 持续监控 DAU
• 分析问卷结果
• 识别活跃用户建 Champion 社群
• 扩展目标人群

关键动作：Step 2 必须由领导发欢迎信——来自高层的信号能显著提高 Day 1 打开率。Step 5 的 Champion 社群是 Journey 的自然延续——完成者是最理想的下一批 Mentor。

Day 1 · Copilot Chat — Get the latest information

Week 1 · Manage My Time

教学目标

用户第一次主动打开 Copilot。体验跨应用数据整合能力。

操作步骤

打开Copilot Chat→输入Prompt→观察结果：Copilot从Email/Chat/Files三个维度汇总→追问一个具体项目

邮件引导语

通过 Copilot Chat 获取本周工作动态。打开 Copilot Chat，用这个 Prompt 了解你本周处理了哪些邮件、聊天和文件。

Prompt："What happened this week? Summarize what I worked on, organized by emails, chats, and files."

教师提示：Day 1最重要——没人打开后面全废。不要纠正Prompt，先让用户"玩"。在群里问"它总结出什么你忘了的事？"

Day 4 · Outlook — Reply to an email

Week 1 · Manage My Time

教学目标

用户用Copilot写出有策略、有语气的邮件回复——不只是"Thank you for your email"。

操作步骤

打开Outlook→选一封需要回复的邮件→点Copilot→输入含语气的Prompt→微调→发送

Day 29 · Dream Big 终极项目 — 完整操作指南

Source: The Great Microsoft 365 Copilot Journey Day 29 project instructions.pdf · 10 pages
全程在一个 Copilot Chat 对话中完成 · 约 20 分钟 · 不可拆分到多个对话（会丢失上下文）

01 The Concept · 点燃创意

自己先想好三要素：常见场所+风格主题+出人意料的地点。然后让Copilot扩展创意。

Copilot技能：Kickstarting creativity · Setting context

Prompt："I'm working on a project that will transform a common place into a new and exciting destination. Give me a creative list of 10 ideas that each include a common place, a creative aesthetic theme, and a creative location like outer space, a fairytale land, or a rainforest."

02 The Context · 定义Copilot角色

给Copilot一个"职位描述"——它将成为你的专属体验设计师。

Copilot技能：Providing context

Prompt："You are an extraordinary experiential designer who can craft immersive destinations. You think creatively, pay attention to every detail, and always deliver unique results."

03 The Exterior · 外观设计

生成外观描述→生成外观图片→下载最喜欢的。

Copilot技能：Defining tone · Inclusions/exclusions · Creating an image

Prompt 1："The project will be to create a [your place] with the theme of [your theme] in [your location]. Create a paragraph with a travel marketing tone. Do not give it a name."

Prompt 2："Create an image that visualizes the exterior of this destination from a distance."

04 The Entrance · 入口设计

入口是第一印象。脑暴5个创意→选一个展开→旅行杂志风格→生成图片。

Copilot技能：Contextual directives · Follow-up · Summarizing · Tone · Image

Prompt 1："Let's think about the entrance. It should embrace the theme and be memorable. Suggest five ideas. This is called The Entrance."

Prompt 3："Rewrite that description in a tone you'd find in a travel magazine. Single paragraph."

Prompt 4："Create an image that visualizes the description of the entrance."

05 The Feature · 核心空间

设计主空间——氛围、座位、灯光。表格格式列出5个空间→选一个→生成图片。

Copilot技能：Formatting · Tone · Creativity · Image

Prompt："We are now working on The Feature. Suggest 5 unique interior spaces. List: ## Name ## Description."

06-08 The Food / Drink / Music · 菜单·饮品·音乐

设计招牌菜单(5道菜)→招牌饮品(5款)→10首歌播放列表。每步生成对应图片。

Copilot技能：Reference context · Table formatting · Brainstorming · Image generation

09-10 The Description & Share · 品牌命名→整合发布

取5个名字+tagline→选一个→旅行杂志三句话描述。最后让Copilot回顾整个对话，输出最终版本，Edit in Pages→插入图片→分享。

Copilot技能：Ideation · Summarizing · Tone · Reference previous work · Creating a Page

最终Prompt："Review everything you wrote. Give me the latest edited copy: Introduction, entrance, feature, food, drink, music, branding. Format: ## Title ## Description"

教师提示：这个项目覆盖Copilot全部核心技能——创意激发、上下文管理、格式化输出、多轮细化、图片生成、文档整合。完成后的作品是学员最好的毕业作品。全部30个Prompt来自官方10页PDF。

Source: The Great Microsoft 365 Copilot Journey Day 29 project instructions.pdf · Pages 1–10 · All prompts verified

Pre/Post Journey Survey & DAU Monitoring 操作指南

Pre-Journey Survey（启动前）

测量用户对 Copilot 的初始感知，建立对照基线。

  • 你对 Copilot 的了解程度？（1-5）
  • 你目前使用 Copilot 的频率？（每天/每周/每月/从未）
  • 你最期待 Copilot 帮你解决什么问题？（开放）
  • 你对 AI 工具的态度？（积极/中立/怀疑/抵触）

工具：Microsoft Forms / SurveyMonkey · 在 Kick-Off 邮件中附链接 · 设置 3 天内截止

Post-Journey Survey（结束后）

在 Day 30 发送，测量变化 + 收集团队反馈。

  • 你对 Copilot 的了解程度？（对比 Pre）
  • 你现在使用 Copilot 的频率？（对比 Pre）
  • Journey 中最有用的 3 个 Prompt？（多选）
  • 你有多大可能向同事推荐？（NPS 0-10）
  • 你对下一期 Journey 的建议？（开放）

配合 Badge 发放一起推送 · 回复率通常比 Pre 高（用户已有参与感）

DAU Baseline Monitoring

在 Setup 阶段从 Microsoft Copilot Usage Report 拉取基线数据。

  • 抓取时间点：Journey 开始前 1 周
  • 核心指标：Daily Active Users (DAU) / Weekly Active Users (WAU)
  • 入口：Microsoft 365 Admin Center → Reports → Copilot Usage
  • 对比节点：W2 / W4 / W6 各拉一次 → 形成趋势图

Champion Community 启动法

Follow Up 阶段：从 Journey 参与者中识别高活跃用户 → 建群。

常见 FAQ

"我没时间"

每个练习 ≤5 分钟。用 Copilot 做你本来就要做的事——不是额外任务，是换个方式工作。

"答案不对"

Copilot 不是搜索引擎——给更多上下文、指定来源、追问"再检查一遍"。

"跟我工作无关"

把 [topic] 换成你正在做的事。Journey 教你"怎么让 Copilot 为你工作"，不是让你学某个具体功能。

"结束后呢？"

进入 Champion 社群进阶内容Agent 共创成为下一批 Mentor。

邮件发送指南 — 30封Daily Email Templates 全解析

Source: The Great Microsoft 365 Copilot Journey · 30 daily email templates (.oft) · All verified against original OFT files

邮件结构（以Day 02为例）

组成部分内容
标题The Great Microsoft 365 Copilot Journey
引导语"Today, you'll learn how to catch up on meetings with Microsoft 365 Copilot. Open a meeting chat in Microsoft Teams, click the Copilot icon and practice this prompt:"
Prompt"Recap the topic of this meeting. What are the key decisions and action items?"
CTALearn more → / See what's ahead →

每封邮件的核心是引导语+Prompt。引导语告诉用户"为什么"和"在哪里"，Prompt告诉用户"怎么做"。

发送节奏建议

邮件时间说明
Daily每天早上 8:00用户到公司第一件事看到
Weekly每周一早上 8:00作为本周概览，内含5天Prompt汇总
Fun Day（周五）周五早上 8:00轻松收尾，鼓励社交分享
Day 01 欢迎可提前一天发预热+确保第一天就有人打开
Day 30 毕业周五下午带Badge附件，鼓励LinkedIn分享

定制化建议

  • 本地化：邮件正文翻译成中文，保留英文Prompt（用户需要复制粘贴到Copilot）
  • 品牌化：替换邮件header图片为莱茵/TÜV品牌
  • 链接替换：aka.ms 短链接替换为你的内部Wiki/Teams频道
  • 签名：加项目负责人签名，增加人情味
  • 追踪：如果邮件系统支持，加UTM参数追踪打开率

Weekly vs Daily — 怎么选？

  • Daily 模式：30封，每封1个Prompt。适合高度投入的种子用户。打开率高但发送量大。
  • Weekly 模式：6封，每封5个Prompt（一周汇总）。适合大规模推广。发送量小但单封信息量大。
  • 混合模式（推荐）：Weekly 作为主力推送 + Daily 用于Fun Day单独发（提升趣味性）
  • 预排程：全部邮件在Setup阶段就排好定时发送，不要手动每天发

邮件发送频率统计：主Journey = 30封Daily + 6封Weekly = 36封 | Chat Edition = 30封Daily + 6封Weekly = 36封 | 如双版本同时跑 = 最多72封。建议双版本先跑Chat Edition（零门槛）→再跑主版（M365全应用）。
