Part 1：What is Microsoft Copilot?

10 min 建立认知——Copilot 到底是什么、不是什么

1.1 先理解一个场景

假设你今天要拜访通威集团采购负责人，推荐 ISO 42001 AI 管理体系培训。过去你要做：

过去（没有 Copilot）现在（有 Copilot）
① 翻 SharePoint 找通威的历史项目资料 → 15分钟① 在 Teams 输入："通威做过哪些项目？我们给他们推荐过什么课程？"→ 30秒
② 找课程目录，挑相关课程 → 10分钟② "ISO 42001 相关的培训课程有哪些？整理一个对比表"→ 20秒
③ 打开 PPT 模板，复制粘贴做方案 → 40分钟③ "基于通威的历史项目和我们的课程目录，生成一份拜访简报PPT"→ 2分钟
④ 写跟进邮件 → 10分钟④ "根据这次拜访简报，起草一封给通威采购负责人的跟进邮件"→ 15秒
总计：约75分钟总计：约3分钟

关键洞察

Copilot 不是替代你的专业判断——它替代的是"找信息、拼资料、写草稿"这些低价值重复劳动。省下来的70分钟，你用在对客户的理解和方案的打磨上。

1.2 Copilot 不是 ChatGPT —— 三张表讲清楚

维度ChatGPTMicrosoft Copilot
知识范围互联网上的公共知识。你问它"通威做过哪些项目"，它只能编。企业内部的 SharePoint 文档、Teams 聊天、邮件、OneDrive 文件 + 公共知识。它能读到你的工作上下文。
数据安全你输入的内容可能被用于训练模型。企业数据上传 = 合规风险。在 Microsoft 365 租户内运行，遵循企业已有的权限体系。你看不到的文件，Copilot 也看不到。
权限体系没有企业权限概念。谁登录谁就能问。继承 Microsoft 365 权限——你只能问你有权限看的文件。June 能访问的合同，Chris 未必能通过 Copilot 查到。
工作流集成独立的聊天界面。回答完就结束了。嵌在 Teams、Word、Excel、PPT、Outlook 里。在开会时问、在写文档时问、在回邮件时问。
行动能力只能生成文本。不能帮你发邮件、建任务、更新表格。可以：发送邮件、创建 Planner 任务、更新 Lists、生成 PPT。不只是"说"，可以"做"。

一句话记住区别

ChatGPT 知道世界，Copilot 知道你的工作。ChatGPT 是"图书馆"，Copilot 是"你的工作助理"——它知道你在做什么、文件在哪、昨天开了什么会。

1.3 Copilot 三个核心能力——不是概念，是你能直接用的

Understand · 理解

Copilot 理解的不只是文字，是工作上下文。它能读懂：

  • 文档内容："这份合同的核心条款是什么？跟上一版改了哪里？"
  • 会议上下文："刚才讨论的三个方案各自的利弊是什么？谁支持哪个方案？"
  • 工作流："这个项目目前卡在哪个环节？下一步需要谁做什么？"

Create · 生成

不只是"写文章"——是基于你的企业数据和模板生成可用的业务文档：

  • 基于一份 Word 报告，自动生成 PowerPoint 演示文稿
  • 基于 Excel 销售数据，生成趋势分析报告
  • 基于会议讨论，生成行动项清单和跟进邮件草稿

Act · 执行

这是 Copilot 区别于所有通用 AI 的核心——它能帮你干活：

  • "根据这封邮件，在 Planner 中创建一个任务，截止日期设为下周五"
  • "把这个频道的本周讨论整理成一个周报，发送到管理层频道"
  • "当有新文件上传到 SharePoint 项目文件夹时，自动提取关键信息并更新 Knowledge Index"

课堂互动（可选，2分钟）

请每人想一个自己上周花了超过30分钟做的"查找+整理"类工作，然后问自己：如果有一个助手能30秒完成这件事，你会省下时间做什么？

Part 2：Microsoft 365 Copilot Work Scenarios

25 min 场景实操——按工作日流程，不是按App列表

不按"Word能做什么、Excel能做什么"来讲——按一个西区同事的真实工作日来走。

Outlook Copilot

场景1：早上一打开邮箱……

真实痛点："30封未读邮件，哪些是真正需要我处理的？上周那个客户的合同讨论，最新的结论是什么？"

操作路径：Outlook → 右上角 Copilot 图标 → 选择 "Summarize this email thread" 或直接输入指令

Summarize the emails I received since yesterday, highlighting any that need my response. Also, catch me up on the email thread about the Tongwei contract.
Copilot 输出示例：
• 2封需要你今天回复：Rina 确认通威拜访时间；Christina 催本周周报
• 通威合同线程最新进展：价格方案已确认（见第3封附件），需周三前发正式版
• 其余28封为CC通知、Newsletter等，无需处理

效率提升：从30分钟翻邮件 → 30秒看清重点。每天至少省20分钟。

更多用法："Draft a reply to Rina confirming Thursday morning for the visit" / "What's the sentiment of this customer's email?" / "Schedule a follow-up reminder for Friday"

Teams Copilot

场景2：上午10点，团队周会

真实痛点："我迟到了5分钟，前面讨论了什么？会议结束后谁记行动项？"

操作路径：Teams 会议 → 顶部 Copilot 按钮 → 选择 "Catch me up" 或输入自定义指令。会后可从 Recap 标签或会议聊天中再次调用 Copilot。

Catch me up. What have I missed?

List all action items with owners and deadlines mentioned in this meeting.
Copilot 输出示例：
前面5分钟讨论了西安公开课排期，Niko 提出需要调整7月底的课程时间。Jiajia 建议改到8月第一周。

行动项：① Rina — 确认通威拜访时间（今天） ② Jerry — 更新成都公开课海报（周四前） ③ Molly — 整理隆基项目进展（周五前） ④ Niko — 确认7月底课程调整方案（周三前）

效率提升：迟到不焦虑+不用手动记行动项。会议结束即有完整纪要，直接分享到频道。

更多用法："What were the pros and cons of each proposal?" / "Did anyone disagree with the timeline?" / "Create a meeting summary and post it to the General channel"

Word Copilot

场景3：下午要交客户方案

真实痛点："下周拜访一家汽车零部件厂商，需要一份针对性的培训方案。但我不确定我们有哪些相关课程，也不知道怎么搭框架。"

操作路径：Word 新建空白文档 → 按 Alt+I 激活 Copilot → 描述需求（含客户行业、感兴趣的课程方向）→ Copilot 生成初稿 → 人工补充价格、讲师、时间等细节。

Draft a training proposal for an automotive parts manufacturer. They are interested in IATF 16949 internal auditor training, VDA 6.3 process audit, and ISO 42001 AI management system awareness. Reference our course catalog and include: company background section, proposed courses with descriptions, timeline, and next steps.
Copilot 生成示例：
完整的方案框架——公司背景占位符、三门课程详细介绍（含课程目标+大纲摘要）、建议4周时间安排、下一步行动。你只需补充：具体价格、讲师名字、拜访日期。

效率提升：从"盯着空白页发愁"→"基于草稿打磨"。写方案的心理门槛从3小时降到30分钟。

更多用法："Rewrite this paragraph to be more formal" / "Translate this proposal into English" / "Summarize this 20-page document into a one-page executive summary"

PowerPoint Copilot

场景4：方案有了，还要做PPT

真实痛点："方案在 Word 里写好了，但还得做一版PPT给客户展示。复制粘贴、调格式、找图标……又得一个小时。"

操作路径：PowerPoint 新建 → Home → Copilot → "Create presentation from file" → 选择刚才的 Word 方案.docx → 描述想要的风格和页数 → 生成后逐页调整。

Create a presentation from [方案.docx]. Make it professional with our company blue style. Include: title slide, agenda, company intro, 3 course detail slides, timeline, and a next steps slide. Add speaker notes for each slide.
Copilot 生成示例：
8页PPT——封面(含标题+日期)、议程、公司介绍(占位符)、3个课程详情页(含课程目标+大纲要点)、时间规划甘特图、下一步行动+联系人。每页附带演讲者备注。

效率提升：从1小时排版→10分钟调整。PPT不再是"体力活"，而是"优化活"。

更多用法："Add a slide comparing IATF 16949 vs VDA 6.3" / "Make the design more visual with icons" / "Generate a handout version with 3 slides per page"

Excel Copilot

场景5：月底了，要看数据

真实痛点："本月的销售Pipeline数据在Excel里，但我不精通数据透视表。哪个行业机会最多？谁的转化率最高？"

操作路径：打开销售数据 Excel → 确保数据格式化为表格(Table) → Copilot 面板 → 用自然语言提问。Copilot 会自动生成公式、图表和数据透视表。

Analyze this sales pipeline data. Show me: which industry has the most opportunities, which team member has the highest conversion rate, what's the total pipeline value by service category, and the month-over-month trend.
Copilot 输出示例：
• 汽车行业机会最多(12个，¥240K)，新能源第二(8个，¥180K)
• Rina 转化率最高(45%)，其次是 Chris(38%)
• 培训服务占总Pipeline 60%，认证服务25%，TVET 15%
• 环比上月：Pipeline总额增长22%（附带趋势图和颜色高亮）

效率提升：不需要学数据透视表和公式。用自然语言就能获得专业级分析。

更多用法："Highlight cells where value is above average in green" / "Create a forecast for next month based on this trend" / "Show me correlations between industry and deal size"

OneDrive / SharePoint Copilot

场景6：知识不会丢在微信里

真实痛点："我记得Niko做过一个类似的汽车客户方案，但文件在哪？微信文件早过期了。翻三个月的聊天记录也找不到。"

操作路径：Teams Copilot Chat → 切换到 "Work" 标签（而非 "Web"）→ 输入问题。Copilot 会在你有权限访问的所有 SharePoint、OneDrive、Teams 文件和聊天中搜索。

Find recent proposals for automotive clients in our SharePoint. Who worked on them? What courses were recommended? Also check if there are any meeting notes related to these projects.
Copilot 输出示例：
在 SharePoint 03_Customer Knowledge Base 中找到3份汽车客户方案：Niko(5月)推荐 IATF 16949+VDA 6.3 组合；Jerry(6月)侧重 TISAX 信息安全；Rina(7月初)做了 ISO 42001+汽车行业结合的方案。3月会议纪要中记录了客户对价格的反馈。

效率提升：知识不再依赖"谁记得"或"微信有没有过期"，而是企业级的"搜得到"。

更多用法："What's our most recent proposal for a new energy client?" / "Show me all files related to ISO 42001 uploaded in the last 3 months" / "Who in the team has worked on TISAX projects?"

Part 2B：官方确认 Copilot 支持应用 · 深度指南

参考 17个应用逐一展开——官方能力、操作方法、西区场景

Microsoft 365 Copilot App

官方定位：普通用户进入 Copilot Chat、Notebooks、内容生成、Agents 和工作资料的统一入口（Microsoft Support）。

核心能力：Chat（通用对话）· Copilot Search（跨应用搜索）· Notebooks（多资料深度分析）· Pages（协作页面）· Create（内容生成）· Researcher（研究助手）· Analyst（数据分析）· Agents 入口。

操作方式：通过 Microsoft 365 应用启动器打开 Copilot App，或在浏览器访问 m365copilot.com。顶部可切换 Work / Web 标签——Work 模式下基于企业数据回答，Web 模式下基于互联网。

西区应用：作为团队每个成员的 AI 工作台——早上打开先问"昨天的邮件和会议有什么需要我处理的？"，然后根据需求进入具体应用。

Copilot Notebooks

官方定位：将多个参考文件组织成 Notebook 后向 Copilot 提问，适合需要整合多份资料进行综合分析的工作（Microsoft Support）。

核心能力：添加参考文件（Word/PPT/PDF/网页等）→ 围绕所有文件统一提问 → 跨文件综合分析。

操作方式：Copilot App → Notebooks → 新建 Notebook → 添加参考文件（从 OneDrive/SharePoint/本地上传）→ 输入问题。Copilot 会基于所有已添加文件的内容生成回答。

西区应用：把通威过去一年的所有项目总结、方案、会议纪要放入一个 Notebook → 问"通威最关注哪些培训方向？我们成功推荐过哪些课程？下次拜访应该优先推荐什么？"

Prompt示例："Based on all the documents in this notebook, summarize the client's key pain points and training preferences."

Copilot Pages

官方定位：AI 驱动的协作页面，支持多人实时编辑并让 Copilot 辅助内容生成和整理（Microsoft Service Description）。

核心能力：AI 辅助内容生成 · 多人实时协作编辑 · Loop 组件嵌入（可同步到 Teams/Outlook/Word）· 结构化页面组织。

操作方式：Copilot App → Pages → 新建页面 → 输入主题让 Copilot 生成初稿 → 邀请同事协作编辑 → AI 持续辅助优化。

西区应用：团队共创"Q3 西区培训推广计划"——June 创建页面、Copilot 生成初稿、Rina/Jerry/Molly 实时补充各自城市的内容、Copilot 最终整理为正式文档。

Word

官方定位：在 Word 中使用 Copilot 起草、补充、改写和总结文档内容（Microsoft Support Draft and add content with Copilot in Word）。

核心能力：多数据源连接（SharePoint/外部API/数据库）· 复杂对话流设计 · Action 定义和执行 · 多 Agent 编排 · 安全与治理策略 · 分析仪表板。

操作方式：访问 Copilot Studio 门户 → 创建 Agent → 定义 Instructions（行为指令）和 Knowledge（知识来源）→ 配置 Actions（可执行的操作，如创建Planner任务、发邮件）→ 设置 Guardrails（行为边界）→ 发布到 Teams/M365 Copilot。

西区应用：June 在 Copilot Studio 中创建 BS.A West 的三个共享业务 Agent——Sales Agent（拜访准备+方案生成）、Project Agent（项目总结+风险分析）、Training Agent（课程推荐+讲师匹配）。这三个 Agent 共享同一套知识底座（六层架构的L1-L3），由团队共同使用和维护。

Work IQ

官方定位：Microsoft 365 Copilot 的企业知识理解引擎——不是独立 App，而是 Copilot 的底层智能层。通过 Graph Connector 接入企业内外的知识源，提供语义搜索、知识关联和智能推荐能力。

核心能力：语义理解（不只是关键词匹配）· 知识关联（"通威"与"新能源行业"自动关联）· Graph Connector 接入（外部系统数据进入M365知识图谱）· 跨系统知识检索。

与Agent的关系：Work IQ 是 Agent 的"大脑"。Agent Builder 和 Copilot Studio 创建的是 Agent 的"外壳"（Instructions + Actions），Work IQ 提供的是 Agent 的"知识"——让它真正理解企业的业务上下文。

西区应用：当 Sales Agent 收到"准备拜访通威"的指令时，Work IQ 从 Knowledge Index 中理解"通威=新能源行业=ISO 42001潜在客户=历史做过3个项目"，然后将这些语义关联传递给 Agent 用于生成拜访简报。

这17个应用的共同使用原则

① 先问再找——不要先翻文件，先问 Copilot。
② Work > Web——工作场景优先用 Work 模式，回答基于企业数据。
③ 权限即边界——你看不到的文件，Copilot 也看不到；Copilot 的回答遵循你的权限。
④ 生成≠终稿——Copilot 帮你跨越"从0到1"，最终判断和质量把关永远是你。

这六个场景的共同底层逻辑

不是让 AI 替代你的判断——是让 AI 帮你跨越"从0到1"的那道坎。空白页→草稿、没看过的邮件→重点摘要、散落的知识→关联推荐。你在做的不是"学会用工具"，是建立新的工作习惯：遇到任何问题，先问 Copilot。

Part 3：Enterprise Knowledge Foundation

10 min 知识底座——为什么没有知识基础，Agent 就是空中楼阁

3.1 一个让你清醒的对比

假设你问一个"通用AI"

"我们有哪些专家能做 TISAX 培训？"

通用AI只能回答：
"TISAX是德国汽车行业信息安全评估标准……"

它不知道你们有谁。

接入企业知识的 Copilot

"我们有哪些专家能做 TISAX 培训？"

Copilot 能回答：
"根据 Knowledge Index，刘老师（重庆）和张老师（成都）均有 TISAX 授课资质。刘老师服务过3家汽车客户，最近一次是2026年5月。"

它查到了你们的数据。

这就是企业知识底座的价值

通用AI把所有人都拉到同一条起跑线——谁都能问、谁都知道。但企业AI的差异化，来自你独有的知识资产：你做过哪些客户、有哪些专家、哪些方案成功过。这些是竞争对手复制不了的。

4.1 先理解什么是 Agent

前面6个场景，都是你主动问，Copilot 回答。Agent 是下一步——它不只是回答，还会主动执行。

Copilot（助手模式）

"帮我查一下通威做过哪些项目。"

"准备拜访通威。"
→ Agent 自动查找通威历史项目 → 匹配相关课程 → 生成拜访简报PPT → 起草跟进邮件 → 在 Planner 创建任务。
→ 你只需要审阅和发送。

4.2 三类 Agent —— 不是每个人都要学开发

Level 1

Pre-built Agent

微软预置，开箱即用

如 Research Agent、Visual Creator Agent。在 Copilot Chat 中直接选用。

适用：所有人

Level 2

自然语言创建，无需代码

适用：业务骨干

Level 3

Copilot Studio

企业级开发平台

复杂业务逻辑、多系统连接、安全合规治理。专业团队负责。

适用：IT / 专业团队

4.3 Agent Builder —— 一个真实演示

假设你（作为业务骨干）想创建一个"培训方案推荐 Agent"：

步骤操作

4.4 Agent 的五要素——缺一不可

要素含义举例（Sales Agent）
Instructions 指令Agent 要做什么、怎么做"当你收到一个客户名称，自动查找该客户的历史项目、匹配推荐课程、生成拜访简报"

4.5 三个 Agent 案例，对应西区业务

Sales Agent

知识源：Customer Object / Project Object / Service Object

能力：客户拜访准备 · 课程推荐 · 方案生成 · 跟进邮件草拟

场景："准备拜访长安汽车" → 自动生成包含历史项目+推荐课程+专家匹配的拜访包

Project Agent

知识源：Project Object / Lessons Learned / SharePoint 项目文件夹

能力：项目状态总结 · 风险分析 · Lessons Learned 自动沉淀

场景："隆基项目本周进展" → 自动汇总项目状态+风险+下一步

Training Agent

知识源：课程目录 / 讲师数据库 / 公开课排期

能力：课程推荐 · 讲师匹配 · 学习路径生成

场景："一个新能源客户想做AI合规培训" → 推荐 ISO 42001 课程+匹配刘老师+建议时间

这三个 Agent 的共性

都是基于同一套知识底座（L1-L3）工作。Sales Agent 查客户、Project Agent 查项目、Training Agent 查课程——但它们共享同一个 L4（Graph 关系网）和 L5（Work IQ 语义理解）。建一次知识，喂三个 Agent。
