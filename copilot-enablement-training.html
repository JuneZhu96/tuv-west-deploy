Part 1：What is Microsoft Copilot?

10 min 建立认知——Copilot 到底是什么、不是什么

1.1 先理解一个场景

假设你今天要拜访通威集团采购负责人，推荐 ISO 42001 AI 管理体系培训。过去你要做：

过去（没有 Copilot）现在（有 Copilot）
① 翻 SharePoint 找通威的历史项目资料 → 15分钟① 在 Teams 输入："通威做过哪些项目？我们给他们推荐过什么课程？"→ 30秒
② 找课程目录，挑相关课程 → 10分钟② "ISO 42001 相关的培训课程有哪些？整理一个对比表"→ 20秒
③ 打开 PPT 模板，复制粘贴做方案 → 40分钟③ "基于通威的历史项目和我们的课程目录，生成一份拜访简报PPT"→ 2分钟
④ 写跟进邮件 → 10分钟④ "根据这次拜访简报，起草一封给通威采购负责人的跟进邮件"→ 15秒
总计：约75分钟总计：约3分钟

关键洞察

Copilot 不是替代你的专业判断——它替代的是"找信息、拼资料、写草稿"这些低价值重复劳动。省下来的70分钟，你用在对客户的理解和方案的打磨上。

1.2 Copilot 不是 ChatGPT —— 三张表讲清楚

维度ChatGPTMicrosoft Copilot
知识范围互联网上的公共知识。你问它"通威做过哪些项目"，它只能编。企业内部的 SharePoint 文档、Teams 聊天、邮件、OneDrive 文件 + 公共知识。它能读到你的工作上下文。
数据安全你输入的内容可能被用于训练模型。企业数据上传 = 合规风险。在 Microsoft 365 租户内运行，遵循企业已有的权限体系。你看不到的文件，Copilot 也看不到。
权限体系没有企业权限概念。谁登录谁就能问。继承 Microsoft 365 权限——你只能问你有权限看的文件。June 能访问的合同，Chris 未必能通过 Copilot 查到。
工作流集成独立的聊天界面。回答完就结束了。嵌在 Teams、Word、Excel、PPT、Outlook 里。在开会时问、在写文档时问、在回邮件时问。
行动能力只能生成文本。不能帮你发邮件、建任务、更新表格。可以：发送邮件、创建 Planner 任务、更新 Lists、生成 PPT。不只是"说"，可以"做"。

一句话记住区别

ChatGPT 知道世界，Copilot 知道你的工作。ChatGPT 是"图书馆"，Copilot 是"你的工作助理"——它知道你在做什么、文件在哪、昨天开了什么会。

1.3 Copilot 三个核心能力——不是概念，是你能直接用的

Understand · 理解

Copilot 理解的不只是文字，是工作上下文。它能读懂：

  • 文档内容："这份合同的核心条款是什么？跟上一版改了哪里？"
  • 会议上下文："刚才讨论的三个方案各自的利弊是什么？谁支持哪个方案？"
  • 工作流："这个项目目前卡在哪个环节？下一步需要谁做什么？"

Create · 生成

不只是"写文章"——是基于你的企业数据和模板生成可用的业务文档：

  • 基于一份 Word 报告，自动生成 PowerPoint 演示文稿
  • 基于 Excel 销售数据，生成趋势分析报告
  • 基于会议讨论，生成行动项清单和跟进邮件草稿

Act · 执行

这是 Copilot 区别于所有通用 AI 的核心——它能帮你干活：

  • "根据这封邮件，在 Planner 中创建一个任务，截止日期设为下周五"
  • "把这个频道的本周讨论整理成一个周报，发送到管理层频道"
  • "当有新文件上传到 SharePoint 项目文件夹时，自动提取关键信息并更新 Knowledge Index"

课堂互动（可选，2分钟）

请每人想一个自己上周花了超过30分钟做的"查找+整理"类工作，然后问自己：如果有一个助手能30秒完成这件事，你会省下时间做什么？

Part 2：Microsoft 365 Copilot Work Scenarios

25 min 场景实操——按工作日流程，不是按App列表

不按"Word能做什么、Excel能做什么"来讲——按一个西区同事的真实工作日来走。

📧 Outlook Copilot

场景1：早上一打开邮箱……

真实痛点："30封未读邮件，哪些是真正需要我处理的？上周那个客户的合同讨论，最新的结论是什么？"

操作：打开 Outlook → 点击 Copilot 图标

Summarize the emails I received since yesterday, highlighting any that need my response. Also, catch me up on the email thread about the Tongwei contract.
Copilot 输出：
• 2封需要你回复：Rina 问通威拜访时间；Christina 问本周周报
• 通威合同线程：最终确认了价格方案（见第3封附件），需要你周三前发正式版

省时：从30分钟翻邮件 → 到30秒看清重点

💬 Teams Copilot

场景2：上午10点，团队周会

真实痛点："我迟到了5分钟，前面讨论了什么？会议结束后谁记行动项？"

操作：Teams 会议中 → Copilot 面板

Catch me up. What have I missed?

List all action items with owners and deadlines mentioned in this meeting.
Copilot 输出：
前面5分钟讨论了西安公开课排期，Niko提出需要调整7月底的课程时间。

行动项：① Rina — 确认通威拜访时间（今天） ② Jerry — 更新成都公开课海报（周四前） ③ Molly — 整理隆基项目进展（周五前）

省时：迟到不焦虑 + 不用单独记行动项

📝 Word Copilot

场景3：下午要交客户方案

真实痛点："下周拜访一家汽车零部件厂商，需要一份针对性的培训方案。但我不确定我们有哪些相关课程，也不知道怎么搭框架。"

操作：Word 新建文档 → 按 Alt+I 或点击 Copilot 按钮

Draft a training proposal for an automotive parts manufacturer. They are interested in IATF 16949 internal auditor training, VDA 6.3 process audit, and ISO 42001 AI management system awareness. Reference our course catalog and include: company background section, proposed courses with descriptions, timeline, and next steps.
Copilot 生成：
一份完整的方案框架——包含公司背景占位符、三门课程的详细介绍、建议的时间安排、以及下一步行动建议。你只需要补上具体价格和讲师安排。

省时：从"盯着空白页发愁"→ 到"基于草稿打磨"，心理门槛大幅降低

📊 PowerPoint Copilot

场景4：方案有了，还要做PPT

真实痛点："方案在 Word 里写好了，但还得做一版PPT给客户展示。复制粘贴、调格式、找图标……又得一个小时。"

操作：PowerPoint 新建 → Copilot → "Create presentation from file" → 选择刚才的 Word 方案

Create a presentation from [方案.docx]. Make it professional, use our company template style. Include: title slide, agenda, 3 course detail slides, timeline, and a next steps slide.
Copilot 生成：
8页PPT——封面、议程、3个课程详情页、时间规划、下一步。带演讲者备注。你只需要调整排版细节，不需要从零开始。

省时：从1小时排版 → 到10分钟调整

📈 Excel Copilot

场景5：月底了，要看数据

真实痛点："本月的销售Pipeline数据在Excel里，但我不精通数据透视表。哪个行业机会最多？谁的转化率最高？"

操作：打开销售数据 Excel → Copilot 面板

Analyze this sales pipeline data. Show me: which industry has the most opportunities, which team member has the highest conversion rate, and what's the total pipeline value by service category.
Copilot 输出：
• 汽车行业机会最多（12个），其次是新能源（8个）
• Rina 转化率最高（45%）
• 培训服务占总Pipeline的60%，认证服务占25%
（附带图表和数据透视表）

省时：不需要学数据透视表，用自然语言就能分析

📁 OneDrive / SharePoint Copilot

场景6：知识不会丢在微信里

真实痛点："我记得Niko做过一个类似的汽车客户方案，但文件在哪？微信文件早过期了。"

操作：Teams Copilot Chat → 选择工作模式（Work tab）

Find recent proposals for automotive clients in our SharePoint. Who worked on them? What courses were recommended?
Copilot 输出：
在 SharePoint 的 03_Customer Knowledge Base 文件夹中找到3份汽车客户方案。Niko 在5月做过一份，推荐了 IATF 16949 + VDA 6.3 组合。Jerry 在6月做过一份，侧重 TISAX 信息安全。

省时：知识不再依赖"谁记得"，而是"搜得到"

这六个场景的共同底层逻辑

不是让 AI 替代你的判断——是让 AI 帮你跨越"从0到1"的那道坎。空白页→草稿、没看过的邮件→重点摘要、散落的知识→关联推荐。你在做的不是"学会用工具"，是建立新的工作习惯：遇到任何问题，先问 Copilot。

Part 3：Enterprise Knowledge Foundation

10 min 知识底座——为什么没有知识基础，Agent 就是空中楼阁

3.1 一个让你清醒的对比

假设你问一个"通用AI"

"我们有哪些专家能做 TISAX 培训？"

通用AI只能回答：
"TISAX是德国汽车行业信息安全评估标准……"

它不知道你们有谁。

接入企业知识的 Copilot

"我们有哪些专家能做 TISAX 培训？"

Copilot 能回答：
"根据 Knowledge Index，刘老师（重庆）和张老师（成都）均有 TISAX 授课资质。刘老师服务过3家汽车客户，最近一次是2026年5月。"

它查到了你们的数据。

这就是企业知识底座的价值

通用AI把所有人都拉到同一条起跑线——谁都能问、谁都知道。但企业AI的差异化，来自你独有的知识资产：你做过哪些客户、有哪些专家、哪些方案成功过。这些是竞争对手复制不了的。

3.3 BS.A West 六层知识平台架构——逐层拆解

L6 · Shared Business AI Agents

Copilot Studio 构建的业务 Agent：Sales Agent / Project Agent / Training Agent

"谁来执行？"——这是团队共享的数字员工，不是各自为战的个人工具

↑ 知识行动

L5 · Enterprise Knowledge Intelligence (Work IQ)

语义理解 · 上下文关联 · 知识推理

"这意味着什么？"——不仅知道"有哪些客户"，还知道"哪些客户最有可能需要 ISO 42001"

↑ 知识理解

L4 · Knowledge Relationship Layer (Microsoft Graph)

身份 · 权限 · 关系网络

"什么跟什么相关？"——通威 → 新能源行业 → ISO 42001 → 刘老师 → 5月项目方案

↑ 知识连接

L3 · Knowledge Object Layer (Microsoft Lists)

Customer Object / Project Object / Service Object / Expert Object

"怎么结构化？"——每个客户一条记录：行业、项目、联系人、推荐课程、历史方案

↑ 知识结构化

L2 · Knowledge Automation Pipeline

Power Automate + AI Processing · 自动提取摘要/标签/分类

"怎么自动生产知识？"——新项目总结上传 SharePoint → 自动提取客户/课程/成果 → 写入 Lists

↑ 知识生产

L1 · Enterprise Content Foundation

SharePoint 文档库 · Teams 文件 · OneDrive · 邮件 · 会议录制

"文档在哪？"——不强行整理，让文件自然沉淀在 M365 里，由上层自动抽取

六层架构的核心设计哲学

底层不动，上层抽。L1 你该怎么存文件就怎么存，不需要改变工作习惯。L2 的自动化管道帮你从文件中提取知识。L3-L6 逐层加工、连接、理解、行动。AI 层可以变（今天用 Copilot，明天换别的），但 L1-L3 的数据层保持稳定——未来无需推倒重来。

Part 4：Copilot Agent & Copilot Studio

10 min 数字员工——从"我问AI"到"AI帮我干活"

4.1 先理解什么是 Agent

前面6个场景，都是你主动问，Copilot 回答。Agent 是下一步——它不只是回答，还会主动执行。

Copilot（助手模式）

"帮我查一下通威做过哪些项目。"
→ Copilot 搜索、列出结果。
→ 你看了之后自己决定下一步。

Agent（数字员工模式）

"准备拜访通威。"
→ Agent 自动查找通威历史项目 → 匹配相关课程 → 生成拜访简报PPT → 起草跟进邮件 → 在 Planner 创建任务。
→ 你只需要审阅和发送。

4.2 三类 Agent —— 不是每个人都要学开发

Level 1

Pre-built Agent

微软预置，开箱即用

如 Research Agent、Visual Creator Agent。在 Copilot Chat 中直接选用。

👤 适用：所有人

Level 2

Agent Builder

自然语言创建，无需代码

用一句话描述你想让 Agent 做什么，指向数据源，即可创建。适合标准化场景。

👤 适用：业务骨干

Level 3

Copilot Studio

企业级开发平台

复杂业务逻辑、多系统连接、安全合规治理。专业团队负责。

👤 适用：IT / 专业团队

4.3 Agent Builder —— 一个真实演示

假设你（作为业务骨干）想创建一个"培训方案推荐 Agent"：

步骤操作
① 打开 Agent Builder在 Copilot Chat 中选择 "Create an agent"
② 用自然语言描述创建一个 Agent，名叫"BS.A West 课程顾问"。当同事询问某个行业的培训需求时，从 SharePoint 课程目录和 Knowledge Index 客户案例中查找匹配的课程和类似客户的成功案例，生成一份培训推荐方案。
③ 指定数据源Add knowledge → SharePoint 文件夹 "02_Products & Delivery Assets" 和 Knowledge Index List
④ 测试"一家新能源汽车零部件厂想做 IATF 16949 内审员培训和 VDA 6.3 过程审核，推荐什么课程？" → Agent 自动搜索匹配并生成推荐
⑤ 发布Publish → 团队成员可以在 Teams 中 @BS.A West 课程顾问 直接使用

你可以马上想的一个Agent

闭上眼睛想5秒钟：你日常工作中，有什么事是"每次都要重复做同样步骤"的？——比如：每次新客户拜访前要查资料、每次项目结束要写总结、每周五要汇总进展。这些都可以变成 Agent。

4.4 Agent 的五要素——缺一不可

要素含义举例（Sales Agent）
Instructions 指令Agent 要做什么、怎么做"当你收到一个客户名称，自动查找该客户的历史项目、匹配推荐课程、生成拜访简报"
Knowledge 知识Agent 从哪里获取信息SharePoint 客户档案文件夹 + Knowledge Index（Customer/Project/Service List）
Connectors 连接Agent 能连接哪些系统Microsoft Graph（查人）、SharePoint（查文件）、Planner（建任务）
Guardrails 边界Agent 不能做什么不透露其他客户的非公开信息、不报价、不承诺交付时间
Governance 治理谁负责管理这个 AgentJune 是 Owner，每月审核 Agent 的知识源是否过时

4.5 三个 Agent 案例，对应西区业务

Sales Agent

知识源：Customer Object / Project Object / Service Object

能力：客户拜访准备 · 课程推荐 · 方案生成 · 跟进邮件草拟

场景："准备拜访长安汽车" → 自动生成包含历史项目+推荐课程+专家匹配的拜访包

Project Agent

知识源：Project Object / Lessons Learned / SharePoint 项目文件夹

能力：项目状态总结 · 风险分析 · Lessons Learned 自动沉淀

场景："隆基项目本周进展" → 自动汇总项目状态+风险+下一步

Training Agent

知识源：课程目录 / 讲师数据库 / 公开课排期

能力：课程推荐 · 讲师匹配 · 学习路径生成

场景："一个新能源客户想做AI合规培训" → 推荐 ISO 42001 课程+匹配刘老师+建议时间

这三个 Agent 的共性

都是基于同一套知识底座（L1-L3）工作。Sales Agent 查客户、Project Agent 查项目、Training Agent 查课程——但它们共享同一个 L4（Graph 关系网）和 L5（Work IQ 语义理解）。建一次知识，喂三个 Agent。
