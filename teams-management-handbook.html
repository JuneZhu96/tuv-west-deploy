Chapter 0：西区的数字化工作方式

过去是这样

  • 文件散落在微信/QQ群，三天后过期找不到
  • 邮件来回转发，版本混乱——"最终版v3_final_修复.docx"
  • 会议讨论完就散了，没人记行动项
  • 跨城市沟通全靠电话，"成都那边什么情况？"
  • 新人入职翻三个月聊天记录补课

今天可以是这样

  • 所有文件放在 Teams，永久留存、版本可追溯
  • 一个频道一个项目，讨论和文件在同一个上下文里
  • 会议自动录制+转录，Copilot 30秒出纪要
  • 重庆/成都/西安在同一个频道里同步信息
  • 新人进Team就能看到全部历史，自助补课

这不是一本"软件说明书"，这是西区 BS.A 团队的协同工作方式。
学完这本手册，你不只会用 Teams——你会知道重庆、成都、西安三地怎么在同一套体系里高效运转。

真实场景：一个客户项目的协同升级

以前是这样

报价方案在微信群里来回传——"XX客户报价_v3_final_改.docx"。会议开完了没人记行动项，一周后忘了谁负责什么。合同审批靠邮件转发，每次都要翻半天找最新版本。

现在是这样

客户项目独立 Teams 频道。所有文件在 Files 里版本自动管理。每次会议 Copilot 转录→30秒出纪要→Planner 分配任务→频道通知全员。报价审批走 Approvals，June 手机上就能批。

效果

文件不再丢、不再过期。新人进频道就能看到全部项目历史，不需要任何人"交代上下文"。行动项有Owner有Deadline有跟踪。每人打开 Teams 就知道自己本周该干什么。

这本手册就是教你搭建这套体系。从今天开始，西区每一个项目都应该这样跑。

1.1 认识 Teams

Microsoft Teams 是 Microsoft 365 中的团队协作中心——把人员、对话、内容汇集到一个统一平台。它不是一个孤立的 App，而是深度整合了聊天、会议、通话、文件协作的综合通信与协作系统。

Teams 的核心概念：

概念说明类比
Team（团队）一群人、对话、文件和工具的集合。通常对应一个组织单元（如"BS.A West"）。公司里的一个部门
Channel（频道）团队内的讨论区，专门针对某个部门、项目或主题。每个 Team 可以有多个 Channel。部门里的专项工作组
Chat（聊天）一对一或群组对话，更私密、更快速。最多 250 人群聊。微信私聊/群聊
Meeting（会议）支持录制、转录、屏幕共享、白板的在线会议。Zoom / 腾讯会议
Files（文件）与 SharePoint 和 OneDrive 深度集成的文件存储与协作。企业网盘 + 协同编辑

1.2 团队与频道设计

Team（团队）是一群人员、对话、文件和工具的集合。一个Channel（频道）是团队内的专门讨论空间。

创建团队的两种方式

方式一：从模板创建

1左侧面板选择 Teams → Create and join teams and channels
2选择 Create team → More create team options
3浏览模板，点击查看每个模板包含的频道和 App，选择 Use this template
4设置团队名称和描述 → 选择 Private（需批准加入）或 Public（组织中任何人可加入）
5选择 Customize 调整频道 → Create

方式二：从零创建

1左侧 Teams → Create and join teams and channels → Create team
2设置名称和描述 → 选择 Private 或 Public
3命名第一个频道 → Create

添加频道

在团队名旁选择 More options（⋯） → Add channel → 设置名称和描述 → 选择频道类型：

频道类型可见范围适用场景
Standard（标准）团队内所有人可访问日常讨论、全员通知
Private（私有）仅特定成员可访问敏感项目、管理层讨论
Shared（共享）可邀请组织外人员跨公司合作项目

添加成员

团队名旁 More options → Add member → 输入姓名添加。

1.3 聊天与协作

一对一聊天

1选择 Chat → New chat 图标
2在 To 字段输入对方姓名
3输入消息 → Send

也可以从频道中点击对方头像打开个人资料卡来发起聊天。

群组聊天

创建方式与一对一聊天相同，只需在 To 字段中添加多个人。群聊最多容纳 250 人。建议给群聊命名（点击群聊顶部→输入名称），方便后续查找。

频道帖子

频道内的对话以"帖子（Post）"形式组织。团队成员可以在频道中：

  • 发起新对话（New post）
  • 回复已有帖子（形成讨论串）
  • @提及（@team 通知整个团队，@某人通知特定人员）
  • 附加文件、图片、链接
  • 使用格式工具（加粗、列表、代码块等）

Catch up（消息追赶）功能

Teams 提供 Catch up 视图帮助快速浏览所有未读和重要消息。每条对话以可滑动的卡片呈现，可以直接阅读、回复、反应，无需来回跳转。处理完所有消息后会显示"You're all caught up"。

1.5 文件协作

Teams 中的文件功能深度集成了 OneDrive for work or school 和 SharePoint。每个频道自动拥有一个 SharePoint 文档库，团队上传的文件实际存储在那里。

文件入口

在 Teams 左侧选择 OneDrive 图标，可看到：

视图内容
Home最近查看/编辑的所有文件
My files个人 OneDrive 中的所有文件和文件夹
Shared与你共享 + 你共享给他人的文件
Favorites已收藏的文件
Recycle bin已删除的文件

快速浏览方式：Browse files by → 按分享者、会议、媒体类型筛选。Quick access → 直接进入特定频道文件。

上传和分享文件

1打开聊天/频道/会议 → 选择 Shared 或 Files 标签
2选择 Upload → 选择文件来源 → 选择文件 → Open
3文件自动上传，团队成员根据权限查看或编辑

从 OneDrive 分享：OneDrive → My files → 悬停文件 → More Actions → Share（直接发送）或 Copy link（复制链接）或 Manage access（调整权限）。

权限说明

Teams 文件的访问权限继承自 Team/Channel 的成员权限 + SharePoint 文档库权限。频道成员默认有编辑权限（除非单独设置只读）。分享给外部人员需单独授权。

1.6 通知与状态管理

通知设置入口

Settings（）→ Notifications and activity（）。可分类型精细控制：

通知类别可选方式
聊天消息Banner and feed / Only show in feed / Off
@提及Banner and feed / Only show in feed / Off
频道帖子Banner and feed / Only show in feed / Off
会议开始提醒开关
会议聊天（已接受）下拉选择通知级别
会议聊天（暂定/无回应）默认：Mute until I join or send a message

全局控制

  • 静音所有通知（通话和会议除外）：Windows 专属开关
  • 播放通知声音：可区分紧急消息和优先联系人
  • 显示消息预览：通知横幅中显示内容片段
  • 通知位置：右下/右上/左下/左上可选
  • 未读活动邮件：定期发送错过了的活动摘要邮件

状态管理

点击头像 → 选择状态：Available（在线）、Busy（忙碌）、Do not disturb（勿扰）、Be right back（马上回来）、Appear away（显示离开）、Appear offline（显示离线）。可设置持续时间（如"1小时后恢复"）。

频道级别的通知

悬停频道名 → More options → Channel notifications → 可选择关注所有新帖或仅关注回复。

沟通工具选择指南：什么时候用什么？

不是所有沟通都适合用 Teams——选对工具，效率翻倍。

场景推荐工具原则
一对一快速确认Teams Chat比邮件快，比微信有记录。私聊不打扰团队信息流。
项目内讨论Teams Channel上下文永久留存。新人进频道就能看到全部历史。
全员通知/公告Channel 帖子 + @team全员可见、不会被群聊淹没。重要通知必须沉淀。
跨城市/跨部门协调Teams Meeting屏幕共享+录制+转录，异地无差别。会后Copilot出纪要。
正式客户沟通Outlook 邮件邮件具有法律效力。外部客户不一定有Teams。
合同/报价审批Approvals + Channel审批流+文件在同一上下文，审批历史可追溯。
紧急事项Teams Chat @某人通知直达对方，响应最快。比打电话好在有文字记录。
个人工作草稿OneDrive + 私聊自己不影响团队信息流。草稿成熟了再发频道。
微信群还保留吗保留，仅用于非工作社交工作内容逐步迁移到Teams。微信=紧急联系+生活。

简单原则：需要留存记录的 → Teams 频道。需要法律效力的 → Outlook。需要快速响应的 → Teams Chat。不需要存、聊完就忘的 → 随意。

本部分目标

系统了解 Teams 中的应用生态，将合适的 App 嵌入西区日常管理流程。Teams 中的 App 分为四类：核心应用（开箱即用）、Microsoft 官方 App（Planner/Approvals 等，多数包含在 M365 订阅中）、第三方 App（经 Microsoft 验证）、自定义 App（组织内部开发）。本教材重点展开 Microsoft 官方 App——零额外成本、原生集成。

2.1 Planner ★★★★★ 核心

任务管理中枢。统一了 Microsoft To Do（个人任务）、Planner（协作计划）和 Project for the web（高级项目）的体验。

核心功能：

  • 看板式任务管理（Buckets + Tasks）
  • 任务分配、截止日期、优先级、附件
  • "My Tasks"视图聚合所有个人和共享任务
  • 频道内嵌 Tab，任务与讨论在同一上下文
  • 任务发布（Task Publishing）：总部向一线推送工作计划

西区场景：客户项目跟进看板、活动筹备检查清单、每人本周任务可视化。

M365 订阅自带

→ 官方教程

2.2 Approvals ★★★★ 重要

电子审批流。直接在 Teams 中发起、跟踪和管理审批请求，数据存储在 Dataverse 中。

核心功能：

  • 创建审批请求（支持附件、多级审批）
  • 审批模板（团队级别 / 组织级别）
  • 实时通知（Banner + Activity feed）
  • 审计追踪（Microsoft Purview 集成）
  • 每个团队最多 400 个模板，每个模板最多 50,000 条请求

西区场景：培训方案审批、报价审批、出差/费用审批。

M365 订阅自带 需 Power Automate 许可

→ 官方教程

2.3 Lists ★★★★★ 核心

结构化列表管理。在 Teams 中创建、共享和跟踪结构化数据，支持多种视图。

核心功能：

  • 自定义字段（文本/数字/日期/选择/人员等）
  • 多视图：表格、日历、卡片、画廊
  • 频道内嵌 Tab，数据与讨论联动
  • 与 SharePoint Lists 数据互通
  • 支持导入 Excel 数据

西区场景：客户信息清单、公开课排期、讲师档期管理、设备清单。

M365 订阅自带

→ 官方教程

2.4 Loop ★★★★★ 核心

实时协作组件。Loop 组件可以嵌入 Teams 聊天、Outlook 邮件、Word 文档中，且在所有位置实时同步。

核心功能：

  • Loop 组件类型：列表、表格、任务清单、段落、投票
  • 一处编辑，处处同步（聊天/邮件/文档）
  • Loop 工作区：独立的协作空间
  • Copilot 可通过 Graph 检索 Loop 内容

西区场景：项目脑暴、周报协同草稿、会议议程实时协作、待办事项共享。

M365 订阅自带

→ 官方教程

2.5 OneNote ★★★★ 重要

团队笔记本。结构化笔记管理，支持分区、页面、手写、录音等。

核心功能：

  • 团队笔记本（Shared Notebook）多人同时编辑
  • 分区组 → 分区 → 页面 三级结构
  • 支持手写、绘图、录音、文件附件
  • 频道内嵌 Tab，笔记与讨论同一上下文
  • Copilot 通过 Microsoft Graph 可检索内容

西区场景：会议记录沉淀、项目知识库、客户访谈笔记、个人工作日志。

2.7 Power Automate ★★★★ 重要

自动化工作流引擎。连接 1000+ 应用和服务，构建跨系统自动化流程。

西区场景：客户跟进提醒自动化、周报信息自动收集、文件审批流转、培训报名→自动发确认邮件。

Part 3：Copilot in Teams —— June 的 AI 增强层

硬伤前置说明

Microsoft 365 Copilot 需要额外许可。目前西区 BS.A 团队中，仅 June 拥有 M365 Copilot license。本章展示 June 能用 Copilot 做什么，以及如何将 AI 产出通过 Teams 分享给团队。团队成员虽然不能直接使用 Copilot，但可以享受 June + Copilot 加工后的成果。

官方许可路径：M365 Copilot add-on license（约 $30/人/月）或通过 Pay-As-You-Go 按量计费。

3.1 会议智能（Copilot in Meetings）

Copilot 在 Teams 会议中使用转录（Transcript）和会议聊天作为数据源，实时回答问题和总结讨论。

会议组织者的控制

会议组织者可在会议选项中设置 Copilot 的使用范围：

设置Copilot 行为
During and after the meeting需要开启转录。Copilot 随转录启动，停止转录则 Copilot 停用。
Only during the meeting使用语音转文字数据（无需转录）。会议结束后，语音数据和 Copilot 交互历史不可用。
OffCopilot 禁用。同时禁用录制和转录以保护敏感信息。

关键：转录是 Copilot 的必需品。选"During and after"时，转录必须先开启。

会议中使用 Copilot

在会议控制栏中打开 Copilot 面板 → 选择 View prompts 获取建议，或直接输入自定义问题：

  • "What ideas were discussed?"（讨论了哪些想法？）
  • "What are the next steps?"（下一步是什么？）
  • "What questions remain unanswered?"（还有哪些问题没解决？）
  • "Summarize what's been discussed so far"（总结目前为止讨论的内容）
  • "List action items with owners"（列出行动项和负责人）

如果你迟到超过 5 分钟且 Copilot 已激活，系统会主动通知并提供"补课"摘要。

会后使用 Copilot

  • 从会议聊天：会议结束 → 聊天右上角 Open Copilot → Copilot 结合转录内容和聊天消息（含会议前 24 小时内的聊天）生成回答。
  • 从 Recap 标签：如果转录已开启，Recap 标签中也可访问 Copilot。
  • 导出回答：Copilot 回答可导出到 Word 或 Excel 供进一步协作（除非会议有敏感度标签或禁用了复制转发）。

注意事项

  • 重复会议中，如果后续场次被转录，之前的 Copilot 对话历史会被清除。
  • "Only during the meeting"模式下，会议结束后 Copilot 交互数据不可恢复。
  • 会议敏感度标签可能阻止 Copilot 回答导出。

3.2 聊天回顾（Copilot in Chats & Channels）

Copilot 可帮助你快速理解聊天和频道中的对话上下文，无需逐条翻阅。

在聊天中使用

  • 快速补课："Catch me up on this conversation"（帮我回顾这段对话）
  • 提取决策："What decisions were made in this chat?"（这段聊天里做了哪些决定？）
  • 找出关键信息："What files were shared in this conversation?"（这段对话分享了哪些文件？）
  • 总结要点："Summarize the key points from this channel discussion"（总结这个频道讨论的关键点）

Copilot 引用来源时会标注具体的消息，方便回溯验证。

3.3 文件问答

在 Teams 中直接让 Copilot 分析和回答关于文件的问题，无需打开文件逐页翻阅。

  • "What changed in the client proposal since last week?"（客户方案上周以来改了什么？）
  • "Summarize the key findings from this report"（总结这份报告的核心发现）
  • "Compare the two versions of this contract"（比较合同的两个版本）
  • "Extract all action items from this meeting's shared documents"（提取这次会议分享文件中的所有行动项）

Copilot 读取文件内容是基于用户已有的文件访问权限。

3.4 跨应用联动

Copilot 在 Teams 中不只是"聊天机器人"——它能串联多个 M365 应用的数据和操作：

联动场景操作示例
Teams → Outlook"Draft a follow-up email based on this meeting"（根据本次会议草拟跟进邮件）
Teams → Word"Create a project summary document from this channel's discussions"（从频道讨论生成项目总结文档）
Teams → Planner"Create Planner tasks from the action items in this meeting"（从会议行动项创建 Planner 任务）
Teams → Loop"Draft a meeting agenda in Loop based on last week's action items"（基于上周行动项在 Loop 中草拟议程）

3.5 Copilot Chat 通用对话

Teams 中的 Copilot Chat App 是独立的通用 AI 对话入口——不只是聊 Teams 内的事，还可以：

  • 研究某个行业趋势、政策法规 → 生成摘要分享到频道
  • 草拟方案框架 → 复制到 Word 继续编辑
  • 翻译和润色文本 → 直接用于邮件或提案
  • 基于工作文件（Word/PPT/Excel/邮件/聊天）回答具体问题

Part 4：西区管理实战 SOP

本部分目标

将 Teams + Copilot 能力映射到西区 BS.A 团队的六个核心管理场景。每个场景标注清楚： June 操作 团队操作 Copilot 加持。

核心原则：June 有 Copilot，团队没有。所以模式是"June + Copilot 产出 → Teams 频道分发 → 团队自助消费"。

场景 1：周一例会

频率：每周一次  |  参与人：全员（重庆/成都/西安）

4Copilot会议中实时转录，June 用 Copilot 提取行动项："List action items with owners and deadlines"
5June会后用 Copilot 生成会议纪要 → 审核 → 发布到 Weekly-Meeting 频道
6团队查看纪要，认领 Planner 中的行动项任务

工具组合

Teams 会议（录制+转录） → Copilot（纪要+行动项） → Planner（任务分配） → 频道帖子（发布+讨论）

场景 3：跨城市信息同步

频率：日常  |  参与人：全员（重庆/成都/西安）

SOP 流程

工具组合

为什么不用微信群？

微信群聊有三个致命问题：(1) 信息流不可检索，3天前的消息找不到了；(2) 文件过期，换个手机就没了；(3) 新人入群看不到历史上下文。Teams 频道+PWA永久留存+结构化+可检索——更适合组织知识沉淀。

Generate a "Friday Brief" covering sales, projects, and team updates.生成一份"周五简报"：销售进展、项目动态、团队更新

西区数字化协同原则

West China Digital Collaboration Principles — 这六条不是"建议"，是西区 BS.A 团队的工作规范。

01

所有项目必须建 Channel

每个活跃客户/项目独立频道。讨论、文件、决策、时间线——全在一个上下文。

02

文件只放 Teams，不发微信

方案、报价、合同——上传频道 Files。版本自动管理，全员可查。微信只发链接。

03

决策必须沉淀到 Teams

会议上做的决定 → 发频道帖子。谁、什么时候、定了什么——三要素写清楚。不靠记忆。

04

会议必须形成 Action Items

每次会议结束 = Action Items 进 Planner。谁做、什么时候交——当场确认。

05

Planner 是唯一任务来源

不靠微信提醒、不靠口头交代。所有任务在 Planner 里创建、跟踪、标记完成。

06

Copilot 负责整理，人负责决策

AI 出纪要、出初稿、出摘要。审核、判断、拍板——永远是人的事。
