Chapter 0：西区的数字化工作方式

过去是这样

  • 文件散落在微信/QQ群，三天后过期找不到
  • 邮件来回转发，版本混乱——"最终版v3_final_修复.docx"
  • 会议讨论完就散了，没人记行动项
  • 跨城市沟通全靠电话，"成都那边什么情况？"
  • 新人入职翻三个月聊天记录补课

今天可以是这样

  • 所有文件放在 Teams，永久留存、版本可追溯
  • 一个频道、一套文件夹，所有讨论和文件在同一个上下文里
  • 会议自动录制+转录，Copilot 30秒出纪要
  • 重庆/成都/西安在同一个频道里同步信息
  • 新人进Team就能看到全部历史，自助补课

这不是一本"软件说明书"，这是西区 BS.A 团队的协同工作方式。
学完这本手册，你不只会用 Teams——你会知道重庆、成都、西安三地怎么在同一套体系里高效运转。

真实场景：客户拜访准备

以前是这样

同事明天要拜访一家新能源汽车客户。翻微信群找资料→翻邮件找方案→翻个人电脑找课程目录。不知道团队有没有做过类似客户、有没有相关专家、有没有现成案例。花半天拼凑，心里还是没底。

现在是这样

在 Teams 打开 Knowledge Index → 按行业筛选"Automotive / Battery"→ 关联的 Service、Standard、Expert、Case 一目了然 → 点击 Reference Files 进入 SharePoint 查看正式资料 → AI 生成拜访简报+推荐产品清单+提问清单+跟进邮件草稿。

效果

不再拼凑碎片信息。每个客户拜访前，知识库自动告诉你"我们做过什么、谁能帮忙、有什么资料可用"。AI 把资料变成可用的简报和清单。新人也能独立完成高质量的客户拜访准备。

这本手册就是教你搭建这套"一个团队、一个工作空间、一个可信知识来源"的数字化工作体系。

1.1 认识 Teams

Microsoft Teams 是 Microsoft 365 中的团队协作中心——把人员、对话、内容汇集到一个统一平台。它不是一个孤立的 App，而是深度整合了聊天、会议、通话、文件协作的综合通信与协作系统。

Teams 的核心概念：

概念说明类比
Team（团队）一群人、对话、文件和工具的集合。通常对应一个组织单元（如"BS.A West"）。公司里的一个部门
Channel（频道）团队内的讨论区，专门针对某个部门、项目或主题。每个 Team 可以有多个 Channel。部门里的专项工作组
Conversation（对话）一对一或群组对话，更私密、更快速。最多 250 人群聊。微信私聊/群聊
Meeting（会议）支持录制、转录、屏幕共享、白板的在线会议。Zoom / 腾讯会议
Files（文件）与 SharePoint 和 OneDrive 深度集成的文件存储与协作。企业网盘 + 协同编辑

1.2 团队与频道设计

Team（团队）是一群人员、对话、文件和工具的集合。一个Channel（频道）是团队内的专门讨论空间。

创建团队的两种方式

方式一：从模板创建

1左侧面板选择 Teams → Create and join teams and channels
2选择 Create team → More create team options
3浏览模板，点击查看每个模板包含的频道和 App，选择 Use this template
4设置团队名称和描述 → 选择 Private（需批准加入）或 Public（组织中任何人可加入）
5选择 Customize 调整频道 → Create

方式二：从零创建

1左侧 Teams → Create and join teams and channels → Create team
2设置名称和描述 → 选择 Private 或 Public
3命名第一个频道 → Create

添加频道

在团队名旁选择 More options（⋯） → Add channel → 设置名称和描述 → 选择频道类型：

频道类型可见范围适用场景
Standard（标准）团队内所有人可访问日常讨论、全员通知
Private（私有）仅特定成员可访问敏感项目、管理层讨论
Shared（共享）可邀请组织外人员跨公司合作项目

添加成员

团队名旁 More options → Add member → 输入姓名添加。

西区 BS.A 架构：单频道 + 文件库

1.3 对话与协作

私聊（Conversation）

1选择左侧 Conversation 标签 → 点击 新建对话 图标
2在 To 字段输入对方姓名
3输入消息 → Send

也可以从频道中点击对方头像打开个人资料卡来发起对话。

群组对话

创建方式与私聊相同，只需在 To 字段中添加多个人。群聊最多容纳 250 人。建议给群聊命名（点击群聊顶部→输入名称），方便后续查找。

频道对话（Conversation）

频道内的对话以"Conversation"形式组织。团队成员可以在频道中：

  • 发起新对话
  • 回复已有对话（形成讨论串）
  • @提及（@team 通知整个团队，@某人通知特定人员）
  • 附加文件、图片、链接
  • 使用格式工具（加粗、列表、代码块等）

Catch up（消息追赶）功能

Teams 提供 Catch up 视图帮助快速浏览所有未读和重要消息。每条对话以可滑动的卡片呈现，可以直接阅读、回复、反应，无需来回跳转。处理完所有消息后会显示"You're all caught up"。

1.5 文件协作

Teams 中的文件功能深度集成了 OneDrive for work or school 和 SharePoint。每个频道自动拥有一个 SharePoint 文档库，团队上传的文件实际存储在那里。

文件入口

在 Teams 左侧选择 OneDrive 图标，可看到：

视图内容
Home最近查看/编辑的所有文件
My files个人 OneDrive 中的所有文件和文件夹
Shared与你共享 + 你共享给他人的文件
Favorites已收藏的文件
Recycle bin已删除的文件

快速浏览方式：Browse files by → 按分享者、会议、媒体类型筛选。Quick access → 直接进入特定频道文件。

上传和分享文件

1打开聊天/频道/会议 → 选择 Shared 或 Files 标签
2选择 Upload → 选择文件来源 → 选择文件 → Open
3文件自动上传，团队成员根据权限查看或编辑

从 OneDrive 分享：OneDrive → My files → 悬停文件 → More Actions → Share（直接发送）或 Copy link（复制链接）或 Manage access（调整权限）。

权限说明

Teams 文件的访问权限继承自 Team/Channel 的成员权限 + SharePoint 文档库权限。频道成员默认有编辑权限（除非单独设置只读）。分享给外部人员需单独授权。

1.6 通知与状态管理

通知设置入口

Settings（）→ Notifications and activity（）。可分类型精细控制：

通知类别可选方式
聊天消息Banner and feed / Only show in feed / Off
@提及Banner and feed / Only show in feed / Off
频道对话Banner and feed / Only show in feed / Off
会议开始提醒开关
会议聊天（已接受）下拉选择通知级别
会议聊天（暂定/无回应）默认：Mute until I join or send a message

全局控制

  • 静音所有通知（通话和会议除外）：Windows 专属开关
  • 播放通知声音：可区分紧急消息和优先联系人
  • 显示消息预览：通知横幅中显示内容片段
  • 通知位置：右下/右上/左下/左上可选
  • 未读活动邮件：定期发送错过了的活动摘要邮件

状态管理

点击头像 → 选择状态：Available（在线）、Busy（忙碌）、Do not disturb（勿扰）、Be right back（马上回来）、Appear away（显示离开）、Appear offline（显示离线）。可设置持续时间（如"1小时后恢复"）。

频道级别的通知

悬停频道名 → More options → Channel notifications → 可选择关注所有新帖或仅关注回复。

沟通工具选择指南：什么时候用什么？

不是所有沟通都适合用 Teams——选对工具，效率翻倍。

一对一快速确认
场景推荐工具原则
Teams 私聊比邮件快，比微信有记录。私聊不打扰团队信息流。
项目内讨论Teams Channel上下文永久留存。新人进频道就能看到全部历史。
全员通知/公告Conversation 消息 + @team全员可见、不会被群聊淹没。重要通知必须沉淀。
跨城市/跨部门协调Teams Meeting屏幕共享+录制+转录，异地无差别。会后Copilot出纪要。
正式客户沟通Outlook 邮件邮件具有法律效力。外部客户不一定有Teams。
合同/报价审批内部审批系统TÜV 莱茵内部有正式审批体系，不走 Teams Approvals。
紧急事项Teams 私聊 @某人
个人工作草稿OneDrive + 私聊自己不影响团队信息流。草稿成熟了再发频道。
微信群还保留吗保留，仅用于非工作社交工作内容逐步迁移到Teams。微信=紧急联系+生活。

简单原则：需要留存记录的 → Teams 频道。需要法律效力的 → Outlook。需要快速响应的 → Teams 私聊。不需要存、聊完就忘的 → 随意。

本部分目标

系统了解 Teams 中的应用生态，将合适的 App 嵌入西区日常管理流程。Teams 中的 App 分为四类：核心应用（开箱即用）、Microsoft 官方 App（SharePoint/OneNote/Planner/Forms 等，多数包含在 M365 订阅中）、第三方 App（经 Microsoft 验证）、自定义 App（组织内部开发）。本教材重点展开 June 的 Teams 中实际可用的 App——零额外成本、原生集成。

2.1 SharePoint ★★★★★ 核心

正式知识资产层。每个 Teams 频道自动拥有一个 SharePoint 文档库——文件上传到 Teams Files，实际存储在 SharePoint。支持版本历史、权限管理、协同编辑。

核心功能：

  • 文档库：按文件夹分类（00-09 十类），团队全员可访问
  • 版本控制：自动保存历史版本，随时回滚
  • 协同编辑：多人同时在线编辑 Word/Excel/PPT，无需来回传文件
  • 权限管理：频道成员默认编辑权限，可设只读或外部共享
  • 元数据与搜索：文件可按客户/行业/服务类型打标签，Copilot 可检索

西区场景：十类文件夹覆盖全部工作——从商机（01）到产品（02）到客户（03）到讲师（04）到项目（05）到素材（06）到会议（07）到管理（08）到流程（09）到团队门户（00）。

2.3 Lists · Knowledge Index ★★★★★ 核心

结构化知识索引。Knowledge Index 不是文件清单，而是 Knowledge Object Registry——每一行代表一个知识对象。

知识对象类型：

  • Service（服务产品）
  • Customer（客户）
  • Partner（合作伙伴）
  • Project（项目）
  • Expert（专家/讲师）
  • Market（市场/行业）

建议字段：Title · Object Type · Summary · Industry Domain · Service Category · Region · Related Customers/Services/Experts · Reference Files（指向 SharePoint）· Owner · Status · Tags / AI Keywords · Last Reviewed

核心理念：文件保存在 SharePoint，通过 Reference Files 字段关联。Knowledge Index 是"找东西的入口"，不是"存东西的地方"。

3.1 Copilot Chat · 通用智能入口

Copilot Chat 是 Microsoft 365 的通用 AI 对话入口——不是某个应用的附加功能，而是贯穿所有工作场景的智能层。

日常工作中 Copilot Chat 能做什么

场景示例
会议智能"总结这次会议的要点和行动项" / "我迟到了，帮我补课" / "还有哪些问题没解决？"
对话回顾"帮我回顾这段聊天的关键决策" / "这个频道最近讨论了什么？"
信息搜索"查一下 ISO 42001 最新动态" / "新能源汽车出口欧盟需要哪些认证？"
文本处理"把这封邮件翻译成英文" / "根据这些要点写一段方案摘要"
文档起草"起草一份客户跟进邮件" / "整理本周工作要点"

会议中使用 Copilot 的关键设置

会议组织者可在选项中设置 Copilot 的使用范围："During and after"（需开启转录，会后可回看）/ "Only during"（无需转录，语音实时处理）/ "Off"（完全禁用）。转录是 Copilot 的必需品——选"During and after"时转录必须先开启。

Copilot Chat 的使用心态

把它当成团队里那个"什么都懂一点的聪明同事"——不是替你决策，而是帮你更快地整理信息、发现盲区、节省时间。关键习惯：养成"先问 Copilot"的条件反射。

3.2 Knowledge Usage · 知识检索与问答

当团队习惯用 Copilot Chat 处理日常工作后，下一步是让它利用团队已有的知识资产——而不仅仅是通用搜索。

知识检索的三个层次

层次数据源典型场景
个人文件OneDrive、最近文件"总结我上周修改的方案"
团队知识SharePoint 文档库"我们有没有新能源汽车网络安全相关课程？"
结构化知识Knowledge Index (Lists)"哪些专家做过 TISAX？""通威做过哪些项目？"

文件问答

  • "这份客户方案的要点是什么？"
  • "合同两个版本之间改了哪里？"
  • "这份报告里提到的所有行动项列出来"
  • "比较这三个方案的差异"

跨应用知识联动

Copilot 不只是"聊天机器人"——它能串联多个 M365 应用的数据：

联动场景示例
Teams → Outlook"根据这次会议草拟跟进邮件"
Teams → Word"从频道讨论生成项目总结文档"
Teams → Planner"从会议行动项创建 Planner 任务"
Teams → SharePoint"找一下最近上传的 TISAX 相关资料"

为什么这一步不能跳过

没有知识基础，Agent 无法发挥价值。这也是六层架构从 L1（Content Foundation）开始的原因——先把文档放好、知识对象建好，AI 才有"可用的脑粮"。先建知识，再建 Agent。

3.3 Agent Builder · 业务人员快速创建

Agent 不是独立应用，而是 Microsoft 365 Copilot 的扩展能力。它运行在 Copilot 生态中，可以理解特定业务场景、访问指定数据源、执行预定义任务。

三类 Agent：按复杂度递进

类型创建方式适用场景创建者
预构建 Agent开箱即用常见业务场景（如 Visual Creator Agent）所有用户
Agent Builder Agent自然语言描述简单 FAQ / 知识检索 / 标准化流程业务人员
Copilot Studio Agent低代码/专业开发复杂业务逻辑 / 外部系统连接 / 企业级部署IT 和专业团队

Agent Builder 的核心价值

业务人员可以用自然语言描述创建 Agent——不需要写代码。例如：

  • "帮我创建一个 Agent，当同事询问课程信息时，从课程目录中查找并推荐合适课程"
  • "创建一个 Agent，帮销售团队在拜访客户前自动准备简报"
  • "建一个 Agent，每周五自动汇总团队的项目进展"

Agent 的组成要素

每个 Agent 都需要五个要素：指令（Instructions）——它要做什么 / 知识来源（Knowledge）——它从哪里获取信息 / 数据连接（Connectors）——它连接哪些系统 / 行为边界（Guardrails）——它不能做什么 / 治理（Governance）——谁负责管理它。

3.4 Copilot Studio · 企业级生产部署

当 Agent Builder 无法满足需求时——比如需要复杂业务逻辑、外部系统连接、或跨系统自动化——就需要 Copilot Studio。

Copilot Studio 的定位

Copilot Chat = 通用智能入口 → Agent Builder = 业务快速创建 → Copilot Studio = 企业级生产部署

Copilot Studio 能做什么（Agent Builder 做不到的）

能力说明
复杂业务逻辑多步骤条件判断、分支流程、动态变量
外部系统连接通过 Connector 连接 SAP、Salesforce、自建 API 等
跨系统自动化Copilot Studio + Power Automate = 端到端业务流程
安全合规治理权限管理、数据保护、审计日志、版本控制
多 Agent 编排多个 Agent 协同工作，各自负责不同领域

与六层架构的关系

Copilot Studio Agent 对应六层架构的 L6（Shared Business AI Agents）——它调用 L5 Work IQ 的知识理解能力，基于 L4 Graph 的关系网络和 L3 Lists 的结构化知识对象，完成跨系统的业务动作。L2（Knowledge Automation Pipeline）则为它持续提供新鲜的结构化知识。

3.5 Enterprise Agent Governance · 治理与规模化

当 Agent 从一个变成多个，从个人使用变成团队共享——治理就不是可选项，而是基础要求。

Agent 治理的五个维度

维度关键问题对应六层架构
知识治理Agent 能访问哪些数据？数据质量如何保证？L1-L3
权限治理谁能创建 Agent？谁能使用 Agent？谁能修改 Agent？L4 (Graph)
行为治理Agent 能执行哪些操作？边界在哪里？L6 (Agent)
质量治理Agent 的回答是否准确？如何评估和改进？L5 (Work IQ)
生命周期治理Agent 从创建、测试、发布、监控到退役的完整流程全六层

关键认知

不是每个人都要学开发 Agent。大多数人停留在 L1-L2（Copilot Chat + 知识检索）就足够高效工作。少数业务骨干可以进阶到 L3（Agent Builder）创建简单 Agent。L4-L5（Copilot Studio + Governance）是 IT 和专业团队的事。整个团队的 AI 能力是有阶梯、有分工的——各司其职，共同进化。

Part 4：西区业务实战 SOP

本部分目标

六个西区 BS.A 核心业务场景，覆盖从获客到交付的全流程。每个场景标注： June 操作 团队操作 Copilot 加持。

场景 1：客户拜访准备

频率：按需  |  场景：明天拜访客户，需要快速拼装资料包

SOP 流程

1团队在 Knowledge Index 按行业/客户名搜索——查有没有做过类似客户
2团队点开关联的 Service（课程/认证）、Expert（讲师）、Case（案例）
3团队点击 Reference Files 进入 SharePoint，下载方案、课程大纲、案例材料
4June用 Copilot 生成拜访简报：客户背景 + 推荐产品 + 提问清单 + 跟进邮件草稿
5团队拜访完成后在 Conversation 回复结果：见了谁、什么反馈、下一步

工具组合

Knowledge Index（客户/服务关联） → SharePoint（正式资料） → Copilot（简报生成） → Conversation（结果沉淀）

Prompt Library

Prompt（英文）说明
Based on this client profile and our service catalog, create a one-page briefing with recommended products and key talking points.基于客户档案和服务目录，生成一页简报
List 5 questions to ask this automotive client about their quality management needs.列出拜访汽车客户的5个提问要点
Draft a follow-up email summarizing our discussion and next steps.草拟跟进邮件

June Tips

拜访前一天花15分钟走完流程。Knowledge Index 能告诉你"我们做过什么、谁做过、有什么资料"——不用拼凑碎片信息。

场景 2：客户项目跟进

频率：持续  |  场景：签约后项目交付，多线并进需要统一跟踪

SOP 流程

1团队在 03_Customer & Partnership Knowledge Base 下为客户创建专属文件夹——方案/合同/会议记录全放一起
2团队项目关键节点在 Conversation 发更新：确认讲师、排定日期、客户反馈
3团队交付文件（课件/证书/评估报告）上传到 SharePoint 客户文件夹
4June每月用 Copilot 拉项目状态：\"Summarize the delivery status of active projects this month\"

工具组合

SharePoint（客户文件夹） → Conversation（节点同步） → Copilot（月度状态汇总）

June Tips

一个客户一个文件夹。所有跟这个项目有关的东西——方案、合同、课件、证书——全放一起。不用翻邮件、不用问"上次是谁跟的"。

场景 3：公开课全流程

频率：持续  |  场景：从排期到交付的公开课运营

SOP 流程

1团队公开课排期统一放在 02_Products & Delivery Assets 文件夹
2团队报名统计用 Excel 表格嵌在 Conversation：课程/日期/已报名/满员否——实时更新
3团队课前通知、课中反馈用 Forms 收集
4团队课后资料（课件/证书/反馈汇总）上传到 SharePoint 对应课程文件夹

工具组合

SharePoint（排期+课件） → Excel（报名跟踪） → Forms（反馈收集） → Knowledge Index（课程对象沉淀）

Tips

每门公开课结束后，在 Knowledge Index 更新：学员数、反馈分、讲师。这是未来"推荐课程给客户"的数据基础。

场景 4：跨城市业务协同

频率：日常  |  场景：重庆客户需要成都讲师，成都商机需要重庆支持

SOP 流程

1团队需要跨城资源时，在 Knowledge Index 查专家（Expert）对象——看讲师所在地、资质、档期
2团队在 04_Tutors & Certification Resources 找讲师资质和联系方式
3团队在 Conversation 发协调请求 → @讲师所在地同事 → 确认档期
4团队客户信息同步到 03_Customer & Partnership Knowledge Base，对方城市同事直接看

工具组合

Knowledge Index（专家对象） → SharePoint（讲师资质） → Conversation（即时协调）

Tips

跨城最怕"信息不对称"——成都不知道重庆有什么讲师。Knowledge Index 解决这个问题：讲师对象里标了所在地、专长、档期，全团队可见。

场景 5：周报

频率：每周四  |  产出：提交给 Christina 的西区周报

SOP 流程

2June用 Copilot 汇总：\"Summarize this week's key sales, delivery, and team updates\"
3June按周报模板整理：Key Updates / New Opps / Product & Operation Support / Market & Competitors / Team Management Support
4June审核定稿 → 提交 → 发到 Conversation 存档

工具组合

Conversation（业务动态） → Copilot（信息汇总初稿） → Word（周报定稿）

June Tips

Copilot 出初稿 → 审核增删 → 10分钟搞定。关键是把模板喂给 Copilot，让它按模板格式输出。

西区数字化协同原则

West China Digital Collaboration Principles — 这六条不是"建议"，是西区 BS.A 团队的工作规范。

01

所有信息进 General，按文件夹归类

一个 Team、一个 General 频道、靠十类文件夹区分场景。不做多频道切碎信息。

02

文件只放 Teams，不发微信

方案、报价、合同——上传频道 Files。版本自动管理，全员可查。微信只发链接。

03

决策必须沉淀到 Teams

会议上做的决定 → 发频道 Conversation。谁、什么时候、定了什么——三要素写清楚。不靠记忆。

04

会议必须形成 Action Items

每次会议结束 = Action Items 进 Planner。谁做、什么时候交——当场确认。

05

Planner 是唯一任务来源

不靠微信提醒、不靠口头交代。所有任务在 Planner 里创建、跟踪、标记完成。

06

Copilot 负责整理，人负责决策

AI 出纪要、出初稿、出摘要。审核、判断、拍板——永远是人的事。
